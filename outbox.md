**Баг РЕАЛЬНЫЙ, но был у Slack, не у GitHub.** Вот что нашёл ресерчер:

**Slack API (HackerOne #1151518):**
- GET /api/conversations.list с невалидным `Authorization: Bearer faketoken123`
- Slack возвращал **200 OK + список публичных каналов** вместо 401
- Причина: кэш API Gateway проверялся ДО auth middleware. Токен не валидировался, если ответ уже был в кэше.
- Висел годами, потому что:
  - Через UI токен всегда валидный
  - Мониторинг смотрел только статус-код (200 = ок)
  - Никто не тестировал с рандомным токеном
- **Это ТОЧНО твой кейс.** Сценарий один в один.
