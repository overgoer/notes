- Уровень: низкий. Отлично для демонстрации «не доверяй статус-коду».

**3. Restful Booker** — ★★★★☆
- Двойное бронирование: два одинаковых POST /booking с одними датами → 200 OK и разные bookingid. Нет проверки уникальности временного слота.
- curl (дважды): `curl -X POST 'https://restful-booker.herokuapp.com/booking' -H 'Content-Type: application/json' -d '{"firstname":"John","lastname":"Doe","totalprice":150,"depositpaid":true,"bookingdates":{"checkin":"2025-06-01","checkout":"2025-06-05"}}'`
- Два 200, два разных id. Должен быть 409 Conflict.
- Уровень: средний. Хорош для обсуждения бизнес-логики.

**4. GitHub API** — ★★★☆☆
- POST /user/repos без токена → 404 вместо 401 (намеренно, для безопасности). Но баг с точки зрения REST.
- curl: `curl -X POST -H "Accept: application/vnd.github.v3+json" https://api.github.com/user/repos -d '{"name":"test"}'` → 404.
- Уровень: средний. Для обсуждения security-by-obscurity.

**5. OpenWeatherMap** — ★★★☆☆
- Запрос погоды для несуществующего города → 200 с пустыми/дефолтными данными вместо 404.
