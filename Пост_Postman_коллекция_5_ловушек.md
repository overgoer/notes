# Postman-коллекция: 5 ловушек для джуна

**Статус:** ✅ Готов к публикации
**Рейтинг GA:** MUST POST
**Формат:** Обучение / Шпаргалка
**Время публикации:** 11:30 МСК

---

Ты импортировал коллекцию Practicum API, выбрал окружение, жмёшь Send — и видишь 401 Unauthorized. Знакомая боль? Самое время разобрать 4 неочевидные ловушки, которые ловят каждого второго джуна на старте.

**1. Токен протух**
В коллекции есть переменная token с Initial Value — старый JWT, который живёт 24 часа. Ты не заметил, и запрос уходит без токена. Решение: дёрнуть /auth/login, получить свежий токен и вписать его в Current Value переменной.

**2. POST /users возвращает 200 вместо 201**
Создаёшь пользователя — сервер отвечает 200 OK. Документация обещает 201 Created. Это баг — разработчик вернул Ok() вместо Created(). Если твой тест проверяет только 200 — ты пропустишь проблему. Правильный тест: `pm.response.to.have.status(201)`.

**3. baseUrl пуст — запрос в никуда**
Не прикрепил окружение к коллекции? Переменная `{{baseUrl}}` станет пустой строкой. Запрос уходит на `https:///users` — 404 или 400. Спасает пре-реквест-скрипт с проверкой `pm.variables.get("baseUrl")` не undefined.

**4. Зелёный тест на статус — ловушка**
В коллекции висит тест «Status code is 200». Сервер возвращает 200, но тело — HTML с ошибкой (перехваченная 404, обёрнутая в 200). Тест зелёный, а данные — мусор. Обязательно валидируй Content-Type (должен быть JSON) и схему ответа.
