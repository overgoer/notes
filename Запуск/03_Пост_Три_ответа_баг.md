# Пост 3 — Интерактив: "Три ответа API. Один — баг"

**Формат:** викторина → результат → CTA на Free Trial
**Дата:** 01.06.2026 (пн)
**Цель:** >10 кликов по ссылке Free Trial

---

Отправляю GET `/api/users?limit=2`.

Получаю три варианта ответа. Один из них — баг.

**Вариант А**
```json
{
  "users": [
    {"id": 1, "name": "Анна"},
    {"id": 2, "name": "Борис"}
  ],
  "total": 10,
  "limit": 2
}
```

**Вариант Б**
```json
{
  "users": [
    {"id": 1, "name": "Анна"},
    {"id": 2, "name": "Борис"},
    {"id": 3, "name": "Виктор"}
  ],
  "total": 10,
  "limit": 2
}
```

**Вариант В**
```json
{
  "users": [
    {"id": 1, "name": "Анна"},
    {"id": 2, "name": "Борис"}
  ],
  "total": 10
}
```

Вопрос: какой ответ неправильный?

Голосование: **A / B / C**
Правильный ответ и разбор — через 4 часа в комментариях.

---

**Через 4 часа (разбор):**

Правильный ответ: **Б**.

Почему:
- Мы запросили `limit=2` — значит, в ответе должно быть 2 пользователя.
- Вариант Б возвращает 3 пользователя. Параметр limit проигнорирован.
- Вариант В пропустил поле limit в ответе — но это не баг, если клиент не зависит от этого поля. Спорно, но не критично.
- Вариант А — корректный ответ.

limit=2 → 3 пользователя = сервер не умеет ограничивать выборку.
На реальном проекте такая ошибка может означать, что пагинация не работает.
Если в API нет пагинации — при 10 000 пользователях сервер ляжет.
