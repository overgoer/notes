# cURL: 3 лайфхака, которые Postman от вас скрывает

**Статус:** ✅ Готов к публикации
**Рейтинг GA:** MUST POST
**Формат:** Обучение / Шпаргалка
**Время публикации:** 11:30 МСК

---

Postman умеет скрывать правду. cURL — нет. Вот 3 лайфхака для быстрой отладки API Practicum, которые выжмут баги, спрятанные под капотом GUI.

**1. Content-Type: ловушка для джуна**
В Postman ты выбрал body = raw/JSON, и он сам подставляет заголовок. Откроешь cURL — и `-d '{}'` автоматом шлёт `application/x-www-form-urlencoded`. Сервер парсит это как плоскую строку — и привет, OOM за 10 минут.
**Решение:** всегда вручную добавляй `-H "Content-Type: application/json"`. Проверить отправленный заголовок: `curl -v ... 2>&1 | grep "> Content-Type"`.

**2. Тайминги, которые Postman скрадывает**
Postman показывает одно число — total time. cURL через `-w` раскладывает по фазам: `time_connect` (TCP), `time_starttransfer` (сервер думал), `time_total`. Если первый байт приходит за 1.2 с, а передача — 0.3 с — это N+1 SQL без индекса.
**Команда:**
`curl -w "connect: %{time_connect}\nstart: %{time_starttransfer}\ntotal: %{time_total}" -o /dev/null -s https://api.practicum.yandex/endpoint`

**3. Пагинация: проверка за один выстрел**
В Postman нужно листать страницы. Через cURL проверяешь границу: `page=999999&limit=10`. Если ответ 200, а тело не пустое — API возвращает бесконечный список или дубликаты. Быстрее — посмотреть заголовки: `curl -I "?page=1" | grep -i "x-total-pages\|link"`.

**Бонус: `curl -I` всегда первым делом**
HEAD-запрос покажет Content-Type, Content-Length, Cache-Control до того, как тащить тело. Если заголовок обещает `application/json`, а в реальности сервер отдаёт HTML — ты узнаешь об этом за 5 мс, а не после скачивания мегабайта.

**Takeaway:** Postman — это фантик. cURL — нож. Если вы джун, который никогда не писал `-w` или не смотрел `-I` — срочно исправляйтесь.
