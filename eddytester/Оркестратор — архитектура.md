## Пайплайн

### 0. IMAP — входные команды
- IMAP → Gmail, INBOX + Sent Mail за 7 дней
- Команды:
  - `тг 1` — одобрить пост
  - `зашквар: X` — заблокировать тему
  - `в пул: X` — добавить тему в пул
  - `бэклог <идея>` — добавить в wishlist
  - `правка: ...` — редакция поста
  - `dev: посмотри/найди X` → RESEARCH (логируется для Researcher)
  - `dev: добавь/установи X` → CODE → PM Agent (GO/NO_GO)
  - `стратегия: ...` — ответ BSA (диалог)

### 1 → 5. Контент-пайплайн (без изменений)
1. Выбор темы из топиков (~40), не в blacklist
2. Ресёрчер: Serper + SearXNG, 3 итерации → brief
3. GA: 5 критериев → MUST POST / MAYBE / SKIP
3.5 RAW WRITER: факты, баги, кейсы (без стиля)
4. STYLE ADAPTER: под голос @eddytester
5. Email: Markdown → HTML, Resend API

## dev: — входящие задачи

```
Email: "dev: ..."
    │
    ▼
  PM Agent: классификация (ключевые слова)
    │                    │
    ▼                    ▼
RESEARCH            CODE
    │                    │
    ▼                    ▼
Лог для           DeepSeek с pm_context.json
Researcher        7 критериев оценки
                  → GO / NO_GO / NEED_MORE_INFO
                    │
                    ▼
              Вердикт в Obsidian:
              eddytester/PM Agent/<дата>_PM_<VERDICT>.md
              + pm_history.json
```

**PM Agent** — `/root/blog-analysis/agents/orchestrator/pm_agent.py`
- Читает `pm_context.json` — миссия, продукты, серверы, репозитории, 7 критериев
- Классифицирует: RESEARCH (посмотри/найди) vs CODE (добавь/установи)
- DeepSeek V4 Flash → JSON-вердикт
- Сохраняет в `pm_history.json` + пишет файл в Obsidian vault

## BSA — Business Strategy Agent

```
BSA (крон: воскресенье 09:00)
    │
    ▼
Читает: pm_context.json + wishlist.json + pm_history.json
        + состояние репозиториев (git log)
    │
    ▼
DeepSeek с полным бизнес-контекстом
→ State of Business (что сделано, gaps, риски)
→ 3-5 Strategic Bets на неделю
    │
    ▼
Письмо в Obsidian + Email
Тема: "[BSA] Стратегический аудит #14.05"
    │
    ▼
      ┌─────────────────────────────┐
      │  Eddy отвечает:             │
      │  "согласен по 1 и 3,        │
      │   2 переформулируй"         │
      └──────────┬──────────────────┘
                 │
    IMAP → тема [BSA] → BSA анализирует фидбек
                 │
                 ▼
            Уточнённая версия
            (2-3 раунда → консенсус)
                 │
                 ▼
      Итог: eddytester/Стратегия/BSA_2026-05-14.md
```

Это не отдельный агент — модуль в оркестраторе. Запускается по крону, живёт в диалоге.

## Obsidian sync

```
Сервер (агенты)  ←→  GitHub (overgoer/notes.git)  ←→  Мак (крон 10 мин)
                                                          ↕
                                                    iPhone (Obsidian Sync)
```

- Агенты пишут вердикты, BSA, обновления → в `/root/obsidian/notes/eddytester/`
- Хелпер `obsidian-sync commit/pull` — git sync на сервере
- Крон на маке каждые 10 мин: stash → pull → pop → add → commit → push
- Obsidian Sync (нативный) на iPhone — независимо, без конфликтов
