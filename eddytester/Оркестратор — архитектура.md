## Пайплайн

### 0. IMAP (Phase 0)
- IMAP → Gmail, INBOX + Sent Mail за 7 дней
- Команды: `тг 1` (одобрить), `зашквар: X` (блок), `в пул: X` (добавить), `бэклог <идея>` (wishlist), `правка: ...` (редакция)

### 1. Выбор темы (Phase 1)
- Из `topics.json` (~40 тем), неисследованная, не в blacklist

### 2. Ресёрчер (Phase 2)
- Serper.dev (Google) + SearXNG fallback (Bing+Mojeek+HN)
- 3 итерации: EN(10) + RU(10) + SO(5) + Habr(5)
- → brief

### 3. GA — Content Guardian (Phase 3)
- 5 критериев: релевантность, боль аудитории, конкретика, обучаемость, хайп
- Вердикт: MUST POST / MAYBE / SKIP
- BEST_ANGLE + POST_IDEA + заголовок

### 3.5. RAW WRITER (Phase 3.5)
- Выжимает из брифа все факты, баги, кейсы, команды
- Без стилевых ограничений — плотный дамп материала

### 4. STYLE ADAPTER (Phase 4)
- Адаптация под голос @eddytester
- НЕТ Situation/Analysis/Verdict/Takeaway/Реалы
- НЕТ длинных тире (только дефис)
- Короткие абзацы, BEST_ANGLE, пояснения терминов, 1-2 ссылки

### 5. Email (Phase 5)
- Markdown → HTML (CSS вёрстка)
- Resend API + SMTP fallback
- Тема: «Дайджест Оркестратора: [тема поста]»

## Архитектура агентов (проектируемая)

```
Email → IMAP (Phase 0)
              │
              ▼
         Оркестратор
         ┌────┴────┐
         ▼         ▼
    Researcher   PM Agent
    (Phase 2)      │
                  ▼
              dev_runner → DeepSeek-TUI
                              │
                          feature-ветка
                              │
                     Email: diff на аппрув
                              │
                          merge (после ОК)
```

**PM Agent** — прослойка между оркестратором и dev_runner. Хранит контекст репозиториев, оценивает реалистичность, диспетчеризует. Не фреймворк — 200 строк Python.

**dev_runner.py** — `/root/blog-analysis/agents/orchestrator/dev_runner.py`
- Создаёт feature-ветку
- Запускает `deepseek exec --auto` (таймаут 600с)
- Исключает `.deepseek/` мусор
- Парсит `[ISSUE]`/`[QUESTION]`/`[SUGGEST]`
- При ошибке без изменений — откатывает ветку
