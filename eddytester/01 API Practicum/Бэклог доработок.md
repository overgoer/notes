## 🔴 P0 — Сделать до следующего запуска

- [ ] #практикум Синхронизировать обещание багов
**Проблема:** В введении сказано «минимум 15 багов, может больше». В финале — «штук 10». Расхождение подрывает доверие.
**Варианты:**
- 🅰 Правим видео — говорим «10+ багов» (перезапись 2-3 фраз, ~30 мин)
- 🅱 Проходим по всем 20 багам в процессе курса (нужно переснимать практические уроки, ~неделя)
- ✅ **Рекомендация: 🅰** — быстрая перезапись введения и финала. 30 минут.

- [ ] #практикум Чек-лист тестирования API — создать и отдать
**Проблема:** Обещан «чек-лист для тестирования любого метода», но как артефакт не отдаётся.
**Что сделать:** Создать .md файл (или PDF) с универсальным чек-листом: заголовки, статус-коды, тело, схема, границы. Отдать ученику в финале.
**Время:** 1 час на составление, 30 мин на вёрстку. Итого ~1.5 ч.

- [ ] #практикум Мост видео → бот
**Проблема:** Ученик после видео заходит в бот, а там другие задачи, другой API. Дезориентирует.
**Варианты:**
- 🅰 Переписать одну задачу в боте под Candidates API (чтобы человек тестировал знакомый эндпоинт)
- 🅱 В финальном видео добавить CTA на бота с пояснением: «а теперь закрепи навыки на других API»
- ✅ **Рекомендация: 🅰 + 🅱** — 3 часа. Задача-знакомство с Candidates API в боте + анонс в финале.

- [ ] #практикум Content-Type — показать до/после на V2
**Проблема:** Первый баг (text/plain вместо JSON) найден. V2 импортирован. Но не показано наглядно, что изменилось — ученик просто видит «работает», не понимая механику фикса.
**Что сделать:** В урок 11 (BUG FIX) добавить блок — скриншот/демо: вот запрос к V1 → text/plain, вот к V2 → application/json.
**Время:** 30 мин — скриншоты и текстовая вставка в видео.

## 🟡 P1 — Докрутить в ближайших итерациях

- [ ] #практикум PATCH как full update — усилить драму
**Проблема:** PATCH в API заменяет ВСЕ поля, хотя документация обещает partial update. Это баг, но в видео нет акцента: «Смотрите, документация говорит одно, код делает другое — это классический баг пропущенной спецификации».
**Что сделать:** Добавить 2-3 минуты в урок 13 (PATCH) с демонстрацией: отправили PATCH только с name → получили null по остальным полям → проверили через GET. Аплодисменты.
**Время:** 1 час (перезапись фрагмента + демо).

- [ ] #практикум Документация vs API — драматизировать
**Проблема:** Расхождение документации и реального API — крючок продукта. В курсе оно есть, но подано буднично, без эмоций.
**Что сделать:** В начале каждого практического урока добавить фразу: «А теперь сюрприз — документация говорит одно, а база данных — другое». Сделать это signature-фразой всего курса.
**Время:** 30 мин — вставки в описания под видео + текстовые карточки.

- [ ] #практикум Разрыв «15 багов» — полный аудит списка
**Проблема:** В стратегии 20 багов в таблице. В видео проходится меньше.
**Что сделать:** Пройтись по гугл-таблице, сверить какие баги показаны в видео, каких не хватает. Если какие-то баги не вошли — добавить как «бонусные задания» (текст в описании видео).
**Время:** 2 часа — аудит + написание бонусных заданий.

## 🟢 P2 — Когда будет время

- [ ] #практикум Чек-лист → pinned-пост в Telegram
**Проблема:** Чек-лист лежит в курсе и не работает как входящий трафик.
**Что сделать:** Выложить чек-лист постами (или PDF в pinned) — люди сохраняют, репостят, приходят.
**Время:** 1 час.

- [ ] #практикум Разнообразие подачи: вопросы к зрителю
**Проблема:** Монотонный формат «я рассказываю». Нет пауз на «а ты как думаешь?»
**Что сделать:** Вставить в каждое видео 1-2 вопроса к зрителю (пауза 3-5 сек). В монтаже — текст на экране.
**Время:** 2-3 часа (пересмотр + правка таймлайна).

- [ ] #практикум Видео → пост-анонсы в Telegram
**Проблема:** Новые видео выходят, но нет контентной связки с каналом.
**Что сделать:** Для каждого видео — пост-анонс в eddytester: «В новом уроке нашли баг, который...» с ссылкой.
**Время:** 30 мин на пост × 16 = 8 часов (разово), потом 30 мин на новый урок.
