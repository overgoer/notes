## 1. Доступ по ключам

| # | Сценарий | Ожидание | Статус |
|:-:|----------|----------|:------:|
| 1.1 | POST /free/api/keys без заголовков | 200 + ключ вида `free-trial-xxxx` + `expires_at` | ⬜ |
| 1.2 | POST /free/api/keys повторно | Каждый раз новый ключ | ⬜ |
| 1.3 | GET /free/api/users с валидным ключом | 200, список пользователей | ⬜ |
| 1.4 | POST /free/api/users с валидным ключом | 201, пользователь создан | ⬜ |
| 1.5 | GET /free/api/users без ключа (без x-fix-bug) | 401 + upsell | ⬜ |
| 1.6 | GET /free/api/users с невалидным ключом | 401 + upsell | ⬜ |
| 1.7 | GET /free/api/users с истёкшим ключом | 403 + "Free trial expired" + upsell | ⬜ |

**Как проверить 1.7 (истёкший ключ):**
```bash
# Пока нет 24 часов — заглянуть в БД и вручную протухнуть
sudo -u postgres psql -c "UPDATE free_api_keys SET expires_at = NOW() - INTERVAL '1 hour' WHERE key = 'free-trial-xxxx';"
```

## 2. Rate limit (10 запросов в минуту)

| # | Сценарий | Ожидание | Статус |
|:-:|----------|----------|:------:|
| 2.1 | 10 запросов подряд с одним ключом | Все 200 OK | ⬜ |
| 2.2 | 11-й запрос с тем же ключом | 429 + `retry_after: 60` | ⬜ |
| 2.3 | Запрос через 60 секунд после 429 | 200 OK (счётчик сброшен) | ⬜ |
| 2.4 | Разные ключи не влияют друг на друга | 10 запросов с ключом A + 10 с ключом B = все 200 | ⬜ |
| 2.5 | Rate limit на GET и POST общий | 8 GET + 3 POST = 429 на 11-м | ⬜ |
| 2.6 | Запрос на /ping не расходует лимит | 100 пингов = все 200 | ⬜ |

**Как проверить:**
```bash
KEY=free-trial-xxxx
for i in $(seq 1 11); do
  CODE=$(curl -s -o /dev/null -w "%{http_code}" -H "X-Fix-Bug: $KEY" http://localhost:3001/free/api/users)
  echo "Запрос $i: $CODE"
done
```

## 6. Безопасность / Edge cases

| # | Сценарий | Ожидание | Статус |
|:-:|----------|----------|:------:|
| 6.1 | SQL-инъекция в name: `"'; DROP TABLE free_users; --"` | 201 (или 500 от Postgres, без дропа) | ⬜ |
| 6.2 | SQL-инъекция в статус: `"?status=minor' OR '1'='1"` | Пустой массив (регистр + невалид) | ⬜ |
| 6.3 | XSS в name: `"<script>alert(1)</script>"` | 201, сохраняется как есть | ⬜ |
| 6.4 | Огромный name (10 000 символов) | 500 или обрезается (смотря что в БД) | ⬜ |
| 6.5 | age как строка: `{"name":"A","age":"25"}` | 500 или 400 (смотря как парсится) | ⬜ |
| 6.6 | age как null: `{"name":"A","age":null}` | 400 или 201 с age=null | ⬜ |
| 6.7 | Content-Type не JSON | 400 (должен быть JSON) | ⬜ |
