## Запрос

```
GET /api/v2/profile?user_id=8841
Header: X-Auth-Token: admin_token_99
```

## Ответ

```json
{
    "id": 8841,
    "username": "tester_pro",
    "email": "test@example.com",
    "balance": 1500,
    "password_hash": "sha256$uI89!jd#k9",
    "roles": ["user"],
    "is_active": true
}
```

---

## ❓ Вопрос пользователю

```
Где критическая уязвимость в этом ответе?

A) balance: 1500 — не должно быть видно
B) password_hash в ответе API
C) roles: ["user"] — должно быть больше ролей
```

**[Кнопки: A | B | C]**

---

## ✅ Разбор (после ответа)

**Правильный ответ: B** 🔥

`password_hash` никогда не должен возвращаться в API response.

**Риск:** Злоумышленник может использовать хеш для:
- Брутфорс-атаки (подбор пароля)
- Pass-the-hash атаки
- Обхода аутентификации

**Как должно быть:** Поле `password_hash` должно быть полностью исключено из ответа.
