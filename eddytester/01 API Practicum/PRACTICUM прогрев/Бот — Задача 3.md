# Бот — Задача 3: Products

**Статус:** ✅ Готово

**Endpoint:** `GET /api/products?status=active&has_discount=true&limit=2`

**Баг:** Фильтры `status` и `has_discount` не работают

## Запрос

```
GET /api/products?status=active&has_discount=true&limit=2
```

## Ответ

```json
{
  "items": [
    {
      "name": "Механическая клавиатура",
      "status": "active",
      "price": 5000,
      "old_price": 7000
    },
    {
      "name": "Игровая мышь",
      "status": "archived",
      "price": 2500,
      "old_price": 2500
    }
  ],
  "total_count": 45
}
```

---

## ❓ Вопрос пользователю

```
Что не так с этим ответом?

A) Фильтр status=active не сработал
B) Фильтр has_discount=true не сработал
C) Оба фильтра не сработали
```

**[Кнопки: A | B | C]**

---

## ✅ Разбор (после ответа)

**Правильный ответ: C**

**Проблема 1:** Второй товар имеет `status: "archived"` при фильтре `status=active`.

**Проблема 2:** У второго товара `price = old_price = 2500` — скидки нет при фильтре `has_discount=true`.

**Последствия:** Пользователи видят неактуальные товары, кликают → 404 или ошибка.

**Как должно быть:** API должен применять все фильтры на стороне бэкенда.
