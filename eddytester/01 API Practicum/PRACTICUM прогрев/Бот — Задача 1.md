**Статус:** ✅ Готово

**Endpoint:** `POST /api/v3/orders/create`

**Баг:** `success: true` при ошибке валидации

## Запрос

```json
{
   "item_id": 404,
   "quantity": 0,
   "promo_code": "WINTER2026"
}
```

## Ответ

```json
{
   "success": true,
   "order_id": null,
   "message": "Quantity must be more than 0",
   "server_time": "2026-09-11T12:00:00Z"
}
```

---

## ❓ Вопрос пользователю

```
Что не так в этом ответе API?

A) status: 200 OK — должен быть 400
B) success: true при ошибке валидации
C) order_id: null — должен генерироваться
```

**[Кнопки: A | B | C]**

---

## ✅ Разбор (после ответа)

**Правильный ответ: B**

`success: true` при сообщении об ошибке — это противоречие.

Клиент не поймёт: заказ создан или нет?

**Как должно быть:**
```json
{
   "success": false,
   "order_id": null,
   "message": "Quantity must be more than 0"
}
```

Или статус `400 Bad Request` вместо `200 OK`.
