# Бот — Задача 4: Cart Item

**Статус:** ✅ Готово

**Endpoint:** `PATCH /api/cart/items/55`

**Баг:** Отрицательное quantity прошло валидацию

---

## Скриншот

![Задача 4](vault/eddytester/01%20API%20Practicum/PRACTICUM%20прогрев/ЗАДАЧА_4.jpg)

---

## Запрос

```json
{
   "quantity": -1
}
```

## Ответ

```json
{
   "cart_id": "abc-123",
   "item_id": 55,
   "new_quantity": -1,
   "total_price": -499.00
}
```

---

## ❓ Вопрос пользователю

```
Что не так в этом ответе?

A) quantity: -1 прошло валидацию
B) total_price: -499.00 — отрицательная цена
C) Оба варианта
```

**[Кнопки: A | B | C]**

---

## ✅ Разбор (после ответа)

**Правильный ответ: C** 🔥

Сервер принял отрицательное количество и посчитал отрицательную цену.

**Риск:** Злоумышленник может:
- «Вернуть» товары и получить деньги
- Манипулировать итоговой суммой заказа
- Сломать отчётность и аналитику

**Как должно быть:** Валидация на бэкенде должна отклонять `quantity <= 0` с ошибкой `400 Bad Request`.
