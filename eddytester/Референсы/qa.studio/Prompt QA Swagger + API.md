# Prompt QA: Swagger + API

**Просмотры:** 5 800

**Формат:** Обучающее видео с демонстрацией промпта

## Субтитры

Как ускорить работу в тестировании с помощью нейросети?

У нас есть ожидаемое поведение (документация в Confluence, дизайн в Figma) и фактическое поведение. Представим, что у вас есть документация в Swagger. Задача — протестировать микросервис.

У меня есть заготовленный промпт. Например, есть микросервис "Bureau" с тремя ручками: GET, POST, ещё GET. Есть схема с полями. Разработчик скинул Swagger, больше ничего не скинул и просит проверить.

**Промпт:**

> "Создай Postman коллекцию для тестирования API. Спецификация Swagger и ссылка на Swagger. Требования: проверь базовые CRUD операции, позитивные и негативные сценарии для каждого эндпоинта, граничные случаи, пустые поля, превышение лимитов, некорректные форматы, тесты аутентификации и авторизации, обязательность и типы полей в запросе и ответе, статус ответа. Для каждого метода создай папку, внутри которой один запрос — одна проверка. Нейминг запросов: метод, путь, сценарий. Выведи JSON коллекции в формате Postman Collection. Описание каждого запроса: что тестирует, что проверяет, примеры тестовых данных в body и инструкцию коллекции. Контекст: это API. Аутентификация с помощью API токена и baseUrl = bureaupokemonbattle.ru."

Результат — JSON, который можно отправить в Postman. Готовая коллекция с тестами, папками, описанием и тестовыми данными.
