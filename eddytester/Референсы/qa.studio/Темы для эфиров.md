# Темы для прямых эфиров — на основе шортсов qa.studio

> **Контекст:** 33 шортса qa.studio проанализированы.
> Из них — 20+ содержательных вырезок с эфиров (не мемы).
> Цель: вытащить темы для эфиров на 20-30 мин.
> Категории: 🧠 полезное | 🔥 интересное | 🤣 смешное | ⚔️ противоречивое | 💼 карьерное

---

## 🔥 ТОП-5 — MUST HAVE (начни с них)

### 1. «С чего начинать тестирование?» — универсальный ответ
**Категория:** 🧠 Полезное | **Рейтинг:** 10/10 | **Влияние на ЦА:** максимальный
**Формат:** Мини-лекция
**Суть:** Два вопроса, которые должен задать себе любой тестировщик перед началом работы: «Что я тестирую?» и «Как я это буду делать?». Разобрать на живых примерах из API Practicum.
**Для ЦА:** Джун + мидл. База, которую многие пропускают.
**Угол:** Покажи, как эти два вопроса работают на реальном API (например, /stores из Food Delivery API).

### 2. «Как учиться тестированию самостоятельно» — путь без ментора
**Категория:** 🧠 Полезное | **Рейтинг:** 9/10 | **Влияние на ЦА:** высокое
**Формат:** Сторителлинг + практика
**Суть:** История реального человека — YouTube, книги, фриланс, стажировка в Infotecs. Без ментора, без курсов за 200к. Разобрать по шагам.
**Для ЦА:** Джун, новички, те кто сомневается «а нужно ли оно мне».
**Угол:** API Practicum как следующий шаг после YouTube-самообучения.

### 3. «Как тестировать оплату» — процесс, а не магия
**Категория:** 🧠 Полезное | **Рейтинг:** 9/10 | **Влияние на ЦА:** высокое
**Формат:** Скринкаст
**Суть:** Тест-анализ → чек-лист → позитивные проверки → негативные. Разобрать на живом примере.
**Для ЦА:** Джун + мидл. Платежи — боль любой команды.
**Угол:** Показать на реальном API. Негативные сценарии с корзиной и заказами — идеально под API Practicum.

### 4. «Как тестировщику развиваться» — документация как суперсила
**Категория:** 🔥 Интересное | **Рейтинг:** 9/10 | **Влияние на ЦА:** высокое
**Формат:** Мини-лекция
**Суть:** Главный двигатель роста — не автотесты, а документация. Умение продавать свою экспертизу. Инициативность.
**Для ЦА:** Мидл, сеньор, те кто застрял на одном уровне.
**Угол:** Спорный тейк — «автотесты не равно развитие» — отлично зайдёт для дискуссии в комментах.

### 5. «Что такое REST на пальцах» — для тех кто боится API
**Категория:** 🧠 Полезное | **Рейтинг:** 8/10 | **Влияние на ЦА:** высокое
**Формат:** Скринкаст с Postman
**Суть:** REST — это не страшно. Свод правил. GET, POST, PUT, DELETE, PATCH. Показать прямо в Postman на живом API.
**Для ЦА:** Джун, начинающие, те кто переходит из мануалки в API.
**Угол:** Идеально для рекламы API Practicum — «хочешь научиться? вот тебе тренажёр».

## 🧠 ПОЛЕЗНОЕ (харды, скилы, инструменты)

### 6. HTTP и HTTPS — разница для тестировщика
**Рейтинг:** 8/10
**Формат:** Скринкаст
**Контент:** HTTPS как «бургер» HTTP + TLS. Шифрование. Зачем это тестировщику.
**Угол:** Показать Wireshark в деле — как выглядит HTTP vs HTTPS трафик.
**Для ЦА:** Джун, начинающие API-тестировщики.

### 7. Как посмотреть трафик через Wireshark — ответ на собесе
**Рейтинг:** 8/10
**Формат:** Скринкаст
**Контент:** Вопрос с реального собеса: «HTTPS безопаснее, а как это протестировать?». Ответ — Wireshark.
**Угол:** Лайфхак для собеседований + демонстрация инструмента.
**Для ЦА:** Джун, мидл, кто готовится к собесам.

### 8. Из чего состоит HTTP запрос и ответ
**Рейтинг:** 7/10
**Формат:** Мини-лекция с Postman
**Контент:** Body, Headers, Status — три поля ответа. Адрес, метаданные, сообщение — три поля запроса.
**Угол:** База, которую нужно знать наизусть. Проверь себя.
**Для ЦА:** Абсолютные новички.

### 9. Синтаксис JSON — 6 типов данных
**Рейтинг:** 7/10
**Формат:** Мини-лекция
**Контент:** String, Number, Boolean, Null, Array, Object. На примерах.
**Угол:** Тест в конце эфира — «угадай тип данных».
**Для ЦА:** Джун, новички.

### 10. Shift Left — что это и зачем
**Рейтинг:** 7/10
**Формат:** Мини-лекция
**Контент:** Тестировщик подключается на ранних этапах. Тестирует требования, макеты. Экономит время и нервы.
**Угол:** Противопоставить «тестирование в конце спринта». Показать на примере, как выглядит Shift Left в API.
**Для ЦА:** Мидл, те кто хочет расти.

### 11. Как тестировать микросервисы — в чём сложность
**Рейтинг:** 8/10
**Формат:** Схема + объяснение
**Контент:** Релизные циклы, логи, Grafana, CI/CD для каждого сервиса. Но тестировать микросервисы легче, чем монолит!
**Угол:** Спойлер: сложности есть, но они организационные, а не твои.
**Для ЦА:** Мидл, сеньор.

### 12. Как микросервисы общаются — Kafka, шина, HTTP
**Рейтинг:** 8/10
**Формат:** Схема + рассказ
**Контент:** Kafka, шина данных, прямые HTTP вызовы. На примере Pokemons.
**Угол:** Показать схему взаимодействия — визуализировать.
**Для ЦА:** Мидл, сеньор.

### 13. Что такое тестирование — база на примере стаканчика кофе
**Рейтинг:** 6/10
**Формат:** Метафора + объяснение
**Контент:** Тестирование = сравнение документации и реализации. Стаканчик кофе как метафора.
**Угол:** Хорошо для сторителлинга, но только как часть большого эфира.
**Для ЦА:** Все.

### 14. Промпт QA для Swagger API — нейросеть пишет Postman коллекцию
**Рейтинг:** 9/10 🔥
**Формат:** Скринкаст
**Контент:** Берём Swagger → кидаем в ChatGPT с промптом → получаем готовую Postman коллекцию. Показать промпт, показать результат.
**Угол:** ХИТ! Идеально для демонстрации «я умею работать с AI». Прямая реклама API Practicum — «автоматизируй рутину, тестируй сложное».
**Для ЦА:** Джун, мидл — ВСЕ.

### 15. Какие нейросети применяют тестировщики
**Рейтинг:** 7/10
**Формат:** Обзор
**Контент:** ChatGPT, DeepSeek, Claude, Copilot. Как подать на собесе — «помощник, а не замена».
**Угол:** Как проходить собес с AI. Правильные формулировки.
**Для ЦА:** Все.

### 16. Виды нейросетей — LLM, код, браузер
**Рейтинг:** 6/10
**Формат:** Обзор
**Контент:** ChatGPT, Claude, Cursor, Copilot, Perplexity.
**Угол:** Можно объединить с темой 15.
**Для ЦА:** Все.

## 💼 КАРЬЕРНОЕ (собесы, рост, деньги)

### 17. Как происходит переход из джуна в мидла — спойлер: никак
**Категория:** ⚔️ Противоречивое | **Рейтинг:** 9/10
**Формат:** Сторителлинг
**Контент:** Чёткого разграничения нет. Задачи дают всем одинаковые. «Ты мидл» может никто не сказать.
**Угол:** Отличная провокация. Комментаторы будут спорить, у каждого своя история. Энгейджмент обеспечен.
**Для ЦА:** Джун, мидл.

### 18. Как новичку заслужить репутацию за 2-3 недели
**Рейтинг:** 8/10
**Формат:** Чек-лист + советы
**Контент:** Самостоятельность, разобраться в проекте, запомнить всех по имени, выполнить первую таску за 3 недели.
**Угол:** Конкретный план действий для новичка на проекте.
**Для ЦА:** Джун (прямое попадание).

### 19. Стоит ли быть одним тестировщиком на проекте — ДА!
**Рейтинг:** 8/10
**Формат:** Советы
**Контент:** Бустер для карьеры. Ты знаешь продукт лучше всех. Хард-скиллы растут быстрее.
**Угол:** Разрушить страх «быть одному». Показать плюсы.
**Для ЦА:** Джун, мидл кто боится брать ответственность.

### 20. Как говорить о зарплате на собесе — стратегии
**Рейтинг:** 8/10
**Формат:** Чек-лист
**Контент:** Разные подходы: не называть первым, узнать вилку, называть чёткую цифру. Личный опыт.
**Угол:** Практический эфир с конкретными скриптами диалогов.
**Для ЦА:** Джун, мидл.

### 21. Почему после собеса не дают обратную связь
**Категория:** ⚔️ Противоречивое | **Рейтинг:** 7/10
**Формат:** Объяснение + психология
**Контент:** Опасно говорить правду. Возраст, дискриминация, законы. Компании боятся.
**Угол:** Вызвать эмпатию к HR и одновременно объяснить, почему это бесит.
**Для ЦА:** Все, кто ищет работу.

### 22. Вакансии для тестировщиков — разбор рынка
**Рейтинг:** 7/10
**Формат:** Обзор + цифры
**Контент:** 1200 вакансий, 600 удалёнка, 500 офис, гибрид, требования.
**Угол:** Свежие цифры, аналитика рынка. Повторять раз в месяц — будет заходить.
**Для ЦА:** Джун, мидл.

### 23. Как объяснить родственникам, чем занимается QA
**Рейтинг:** 6/10
**Формат:** Сторителлинг
**Контент:** «Мы с командой разрабатываем приложение. Моя зона — качество продукта».
**Угол:** Можно сделать лёгкий, смешной эфир.
**Для ЦА:** Все (но низкий приоритет).

## 🧪 ПРОТИВОРЕЧИВОЕ (для дискуссий и энгейджмента)

### 27. «Тестировщик» vs «QA» — кто ты на самом деле
**Рейтинг:** 9/10 | **Потенциал хайпа:** 🔥🔥🔥
**Формат:** Полемика
**Контент:** Русское «тестировщик» против англ. QA Engineer. Слово определяет мышление. «Я тестировщик» = «я проверяю», «Я QA» = «я отвечаю за качество с первого дня».
**Угол:** Спорный тейк — соберёт кучу комментов. Каждый считает себя правым.
**Для ЦА:** Все уровни.

### 28. Мануальщик vs Автоматизатор — кто кому нужен
**Рейтинг:** 8/10 | **Потенциал хайпа:** 🔥🔥
**Формат:** Полемика
**Контент:** Когда нанимать автоматизаторов? Только когда продукт устаканился. Ручные тестировщики пишут тест-кейсы, автоматизаторы их покрывают.
**Угол:** Разрушить миф «мануальщик — это прошлый век».
**Для ЦА:** Джун (оправдание выбора), мидл.

### 29. Автотесты НЕ равны развитию — тейк
**Рейтинг:** 9/10 | **Потенциал хайпа:** 🔥🔥🔥
**Формат:** Полемика
**Контент:** Развитие ≠ автотесты. Документация, инициативность, продажа экспертизы — вот что реально двигает карьеру.
**Угол:** Спорный тейк — соберёт хейт и поддержку. Отличный энгейджмент.
**Для ЦА:** Мидл, сеньор, те кто думает «пойду в автотесты, буду больше зарабатывать».

### 30. «Пустая корзина → 201» — как выглядит реальный баг
**Рейтинг:** 7/10
**Формат:** Разбор бага
**Контент:** Примеры реальных багов из реального API (Food Delivery). Почему они возникают, как их находить.
**Угол:** Тизер контента про API Practicum. Показать, что баги бывают неочевидные.
**Для ЦА:** Джун, мидл.
