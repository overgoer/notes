# Interaction of microservices

**Просмотры:** 12 000

**Формат:** Вырезка с эфира — объяснение на примере Pokemons

## Субтитры

Как мы считаем рейтинг? Есть микросервис рейтинга. Чтобы рейтинг посчитать, рейтинг дёргает ручку лавки (собирает оплаты), дёргает ручку монолита (собирает битвы — они влияют на рейтинг), дёргает ручку GET /bureau (информация по скачиваниям). Всю информацию обратно получает, варит и выдаёт рейтинг тренеров на сайте покемона.

**Как микросервисы взаимодействуют друг с другом?**

1. **Через Kafka** — если есть большая нагрузка. Один микросервис сообщение сгружает в Kafka, другой забирает из Kafka. Это **асинхронная передача данных**.
2. **Шина данных (Bus)** — старая технология, где вместо Kafka используется ещё один микросервис-шина, и все сообщения идут через неё.
3. **Напрямую** — синхронные HTTP вызовы.
