## Агенты Фазы Г

### 1. API Scout (Practicum Developer)
- **Роль:** читает код v0-test-api (v1 и v2), BACKLOG.md, DOCS.md
- **Что делает:**
  - Вытягивает безопасные для публикации инсайты (архитектура, подходы, технологии)
  - Оценивает какие темы постов релевантны продукту
  - Генерирует **бэклог задач на практикум** — какие баги/фичи добавить, исходя из тем постов
- **Контекст:** код практикума, текущий BACKLOG, DOCS, ARCHITECTURE

### 2. Free API Developer
- **Роль:** развивает free-trial-api, поддерживает его контент-способности
- **Что делает:**
  - Оценивает какие фичи free API нужны для контента (инста, ютуб, эфиры)
  - Предлагает новые эндпоинты для free API
  - Следит чтобы free API оставался полезным для демо, но не заменял продукт
- **Контекст:** код free-trial-api (server.js + middleware.js), требования контент-плана

### 3. Bot Developer
- **Роль:** развивает @api_practikum_bot, оптимизирует воронку
- **Что делает:**
  - Анализирует какие новые задачи/сценарии добавить в бота
  - Синхронизирует контент постов с воронкой бота
  - Предлагает улучшения конверсии (free API upsell → бот → продукт)
- **Контекст:** код бота (main.py), структура воронки, FSM состояния

### 4. Review Agent (надстройка)
- **Роль:** получает задачи от 1-3, оценивает полезность, сложность, реалистичность
- **Формат оценки:**
  - `Полезность:` 1-10 (сколько студентов/подписчиков это затронет)
  - `Сложность:` 1-10 (часов на реализацию)
  - `Спойлер-риск:` красный/жёлтый/зелёный
  - `Вердикт:` делать / не делать / отложить

## Поток данных

```
Пост (оркестратор) → API Scout → бэклог задач практикума
                      Free API Dev → новые эндпоинты для контента
                      Bot Dev → улучшения воронки

                      ↓
                   Review Agent → приоритизация → BACKLOG.md
```

Оркестратор пишет пост → агенты анализа понимают «а что из этого можно добавить в продукт?» → генерируют задачи → Review оценивает → BACKLOG.md обновляется.
