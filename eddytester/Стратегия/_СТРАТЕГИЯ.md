## 2. Бетсы (что и почему)

### Bet #1: Practicum — единственный P0
**Статус:** Выполнение буксует на шаге "платёжка + триал"
**Критический путь:**
1. Free Trial починен (Postgres auth, /free/api/users 404)
2. Платёжка подключена
3. Флоу пройден лично: триал -> регистрация -> оплата -> доступ -> задачи
4. Первый анонс в Telegram
5. Первые продажи

### Bet #2: Контент = трибуна для Practicum
**Формула:** 50/30/20 (вовлечение / обучение / продажа)
**Фильтр 3 вопросов перед публикацией:**
1. Заставит остановиться и прочитать?
2. Вызовет реакцию?
3. Ведёт к Practicum (прямо или косвенно)?
Если нет хотя бы на один — пост не выходит.

### Bet #3: Вовлечение через механику
**Правило:** каждый пост содержит минимум одно из: вопрос-фактчеллендж, механику (угадай/исправь/да-нет), призыв с низким порогом, прямой CTA.

### Bet #4: 3 поста/нед — жёсткая норма
Пн, Ср, Пт. Пропустил -> переносишь, не дублируешь. 2 из 44 готовых + 1 новый.

### Bet #5: Instagram на паузу
Условия разморозки: выручка Practicum > 50 000 руб + 3-4 поста/нед в TG стабильно месяц + Free Trial работает.

## 3. Целевая аудитория

**Основная:** Junior QA и те, кто хочет войти в тестирование.
Им не хватает практики API-тестирования для трудоустройства.
Хотят показать на собеседовании: "я тестировал реальный API, находил баги, писал автотесты".
Готовы платить за практический опыт.

**Вторичная:** Middle QA (реже), студенты курсов (Яндекс.Практикум, OTUS, Stepik).

**Ключевой инсайт:** продукта такого нет на рынке. Practicum — единственный,
где можно реально искать баги и писать тесты, а не читать теорию.

## 5. Контент-стратегия

### Голос канала
- Экспертиза с иронией. Не хайп, не копирование конкурентов.
- Глубокие разборы багов, чек-листы, инструменты.
- Личный опыт укрепляет комьюнити.
- Пост без вопроса/CTA к аудитории — мёртвый пост.

### Форматы
- Вовлечение (50%): угадайки, да/нет, холивары, вопросы с подвохом
- Обучение (30%): шпаргалки, баг-репорты, разборы, VS Разработчик
- Продажа (20%): прямые CTA на Practicum, кейсы из Practicum

### Пул постов
44 готовых поста в архиве. Правило: 2 из 44 + 1 новый (горячая тема) в неделю.
Пост готов = пост публикуется. Не редактировать бесконечно.

### Время публикации
- 18:30 МСК — основные (вовлечение, продажа, сериалы)
- 11:30 МСК — короткие (шпаргалки, баг-репорты)

## 6. Продуктовая стратегия

### Practicum (v0-test-api)
- REST API-симулятор с преднамеренными багами (v1 — баги, v2 — фикс)
- Стек: Node.js + Express + PostgreSQL, порт 3000 на Timeweb
- 5 эндпоинтов: GET /, GET /:id, POST /, PATCH /, DELETE /
- Swagger документация, Postman коллекция
- Статус: API работает, монетизация не запущена

### Free Trial API
- Урезанная версия Practicum для промоушена (2 эндпоинта, rate limit)
- Статус: РАБОТАЕТ
- Критический блокер для всей воронки

### Telegram Bot (@eddytester bot)
- Статус: работает (PM2: tg-bizzy)
- Интерфейс для Practicum: выдача ключей, обратная связь
