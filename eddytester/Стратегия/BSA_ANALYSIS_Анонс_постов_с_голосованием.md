# Анализ: Анонс постов на неделю с голосованием

## Вопрос
Стоит ли ввести еженедельный анонс постов (на неделю/месяц), чтобы подписчики ждали посты и голосовали за те, что хотят увидеть раньше?

---

## 1. Текущий контекст (факты)

| Параметр | Значение |
|----------|----------|
| Подписчики TG | ~1 500 |
| Средний охват | 300-500 (20-33%) |
| Частота (план) | 3 поста/нед — Пн, Ср, Пт |
| Готовых постов | 44 (можно публиковать хоть сейчас) |
| Тем от ресерчера | 17 |
| % постов с комм. | 56% (выше рынка) |
| % мёртвых постов | 44% (каждый второй) |
| Продукт | Practicum — P0, продажи не запущены |
| Free Trial | Сломан (Postgres auth, /free/api/users 404) |

**Ключевой контекст:** стратегия v6.0 только что принята. Система 3 поста/нед — в стадии внедрения. Practicum не запущен.

---

## 2. Что даёт анонс с голосованием (потенциальные плюсы)

### 2.1. Вовлечение
Голосование — механика вовлечения. Подписчик чувствует, что на его мнение влияет на контент. 
- По данным гайда по вовлечению: опросы про Practicum собрали 25+ комментов.
- Голосование — низкий порог входа (1 клик).

### 2.2. Предсказуемость
Подписчики знают, что выйдет в Пн, Ср, Пт. Это формирует привычку.
- Конкуренты (Лебедев) публикуют ежедневно в одно и то же время — подписчики знают, когда ждать пост.

### 2.3. Данные об интересах
Голосование покажет, какие темы реально интересны аудитории.
- Можно отсеять темы, которые никому не нужны.
- Можно найти «тёмных лошадок» — темы, которые набрали голоса, но ты бы их не написал.

---

## 3. Почему НЕ СТОИТ (аргументы против)

### 3.1. Аудитория слишком мала для meaningful голосования
При охвате 300-500 даже 50 голосов — 10-17% от охвата.
**Реалистичный сценарий:** 10-25 голосов. Это демотивирует и автора, и подписчиков («канал мёртвый»).

### 3.2. 44 поста уже готовы — не нужно голосование, нужно исполнение
У тебя 44 поста, которые можно публиковать прямо сейчас. 
- Голосование — это замедление. Ты выбираешь, о чём писать — но писать уже не надо, всё уже написано.
- **Парадокс:** голосование предполагает, что ты пишешь под запрос. Но у тебя уже есть готовый контент на 15 недель вперёд.

### 3.3. Анонс = дополнительный контент, которого нет в плане
3 поста/нед — жёсткая норма. Анонс — это 4-й пост (или замена одного из трёх).
- Если заменяешь — теряешь 1 из 3 готовых постов.
- Если добавляешь — нарушаешь систему (стратегия v6.0: «3 поста/нед, не 2, не 4»).

### 3.4. Система только внедряется — не усложняй
Стратегия v6.0 принята **сегодня**. 
- Спринт 18-24 мая: P0 — починить триал и опубликовать первый пост. Не «добавить голосование».
- **Правило:** не добавлять новые элементы, пока текущая система не работает стабильно 2-3 недели.

### 3.5. Practicum — P0. Не контент-календарь
Все усилия должны быть направлены на запуск Practicum.
- Голосование за темы постов — это про контент, а не про продукт.
- **Воронка:** контент → триал → покупка. Если триал сломан — контент не конвертируется.
- Часы, потраченные на настройку голосования, лучше потратить на починку триала.

### 3.6. Риск разочарования
Подписчики проголосовали за тему «Х». Ты написал пост. Он собрал 0 комментариев.
- **Результат:** подписчики разочарованы («наше мнение не работает»), автор демотивирован.
- Лучше сначала понять, что реально работает, через A/B тестирование форматов, а потом спрашивать мнение аудитории.

## 4. Альтернативы (что сделать вместо этого)

### Вариант A: Анонс без голосования (рекомендуется)
Просто публикуй расписание на неделю в закреплённом сообщении или stories.
- **Что даёт:** предсказуемость, привычка, без риска пустого голосования.
- **Как сделать:** 1 сообщение в Пн утром: «На этой неделе: 
  - Пн 18:30 — POST без тела (угадай баг)
  - Ср 11:30 — Content-Type text/plain (баг-репорт)
  - Пт 18:30 — Сериал 1/10: Мне дали API интернет-магазина»
- **Оценка усилий:** 5 минут в Пн утром.

### Вариант B: Отложить голосование на 1 месяц
Запустить голосование, когда:
1. 3 поста/нед стабильно работают 3-4 недели.
2. Охват >600 (цель через месяц).
3. Есть первые результаты по форматам (какие работают, какие нет).
4. Practicum запущен, есть хотя бы 1 продажа.

**Условия запуска голосования (checklist):**
- [ ] 3 поста/нед в течение 3 недель подряд
- [ ] Средний охват >600
- [ ] Practicum запущен (есть работающий триал + платёжка)
- [ ] Есть данные по топ-3 форматам (что работает лучше)
- [ ] Готов шаблон голосования (опрос в TG)

### Вариант C: Голосование в рамках Practicum (не для контента)
Вместо голосования за темы постов — голосование за **функции Practicum**.
- «Какой баг добавить в Practicum следующим?»
- Это решает две задачи: вовлечение + развитие продукта.
- И работает как CTA: голосование = участие в жизни Practicum.

## 5. Конкретная рекомендация

### 🟢 Рекомендация: НЕ ДЕЛАТЬ АНОНС С ГОЛОСОВАНИЕМ СЕЙЧАС

**Почему:** 
1. 44 поста готовы — нечего выбирать. Нужно публиковать, а не голосовать.
2. Аудитория (1.5K, охват 300-500) слишком мала для meaningful голосования (10-25 голосов).
3. Система 3 поста/нед только внедряется — не усложнять.
4. Practicum — P0. Починка триала и запуск продаж важнее контент-экспериментов.

### ЧТО ДЕЛАТЬ ВМЕСТО:

1. **Вариант A (сразу):** Публикуй расписание на неделю в Пн утром. Без голосования. Просто анонс. 5 минут.
2. **Вариант B (через месяц):** Пересмотреть голосование, когда 3 поста/нед стабильно, охват >600, Practicum запущен.
3. **Вариант C (альтернатива):** Голосование за функции Practicum — вовлечение + развитие продукта.
