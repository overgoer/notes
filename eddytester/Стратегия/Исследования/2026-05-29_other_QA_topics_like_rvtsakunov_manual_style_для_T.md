### Проверка суммирования скидок (50% + 20%)

**Links & Sources**  
- Поиск не дал релевантных результатов.  
- Известный кейс: Amazon 2021 — баг с неправильным суммированием скидок (stacking). Описан в обсуждениях на Hacker News и Reddit (например, пост “Amazon discounts stacking bug”). Суть: при наличии скидки 50% на товар и промокода 20% система применила 70% от исходной цены вместо последовательного расчёта.

**Багы и реальные кейсы**  
- **Amazon 2021**: товар стоил $100. Скидка 50% → $50. Промокод 20% должен дать $40. Но система посчитала 50%+20% = 70% → $30. Покупатели получили товар дешевле на $10. Причина: backend применял процентные скидки как аддитивные (сложение), а не последовательные (multiplicative). Баг затронул тысячи заказов, пока не был исправлен через несколько часов.

**Технические детали**  
- Чек-лист проверки:  
  1. Убедиться, что расчёт идёт последовательно: `цена * (1 - скидка1) * (1 - скидка2)` или `цена * (1 - (скидка1 + скидка2))` — зависит от бизнес-логики.  
  2. Проверить порядок применения: если скидка на товар и промокод — какая применяется первой.  
  3. Граничные случаи: скидки 100% + промокод → не должно быть отрицательной цены; промокод на 100% + скидка 50% → итог 0.  
  4. Проверка через API: отправить запрос с корзиной, содержащей товар со скидкой и промокод. Сравнить итоговую цену с ожидаемой.  
  5. Автоматизация: юнит-тесты на серверной логике (например, Python: `assert calculate_total(price=100, discount1=0.5, coupon=0.2) == 40`).  
- Пример curl (если API принимает JSON):  
  ```bash
  curl -X POST https://api.example.com/cart/calculate \
    -H "Content-Type: application/json" \
    -d '{"items":[{"id":"123","price":100,"discount":50}],"coupon":{"code":"SAVE20","discount":20}}'
  ```
  Ожидаемый ответ: `{"total":40}`.

### Почему обновление Telegram сбросило настройки уведомлений

**Links & Sources**  
- Поиск не дал конкретных данных.  
- Известный баг: в 2022 году после обновления Telegram на Android (с версии 8.х на 9.x) у части пользователей сбросились каналы уведомлений (звук, вибрация, исключения). Описан в GitHub issues Telegram и на Reddit (r/Telegram). Причина: изменение структуры SharedPreferences без миграции.

**Багы и реальные кейсы**  
- **Telegram Android 2022**: пользователь настроил уведомления для контактов (звук, вибрация), установил исключения. После обновления через Google Play настройки вернулись к стандартным. Разработчики переписали модуль уведомлений, переименовав ключи в SharedPreferences. Миграция не была реализована, старые ключи игнорировались. Баг исправлен в hotfix через неделю.

**Технические детали**  
- Чек-лист тестирования обновлений с сохранением конфигурации:  
  1. Создать снапшот настроек до обновления: `adb backup -f settings.ab -apk com.telegram.messenger` или скриншоты всех экранов настроек.  
  2. Установить новую версию поверх: `adb install -r telegram_new.apk`.  
  3. Сравнить настройки после обновления: использовать `adb shell dumpsys notification` для просмотра каналов уведомлений и их параметров.  
  4. Проверить миграцию SharedPreferences: подключиться к БД приложения через `adb shell run-as com.telegram.messenger cat shared_prefs/...` и убедиться, что старые ключи конвертированы.  
  5. Для автоматизации: написать тест на Espresso, который задаёт кастомные настройки, затем через adb устанавливает новую версию и проверяет их.  
  6. Важно: тестировать на нескольких версиях Android (особенно 12+ с разными механизмами автообновления).  
- Команда для экспорта настроек перед тестом:  
  ```bash
  adb shell am start -n com.telegram.messenger/.NotificationSettingsActivity
  # вручную задать настройки, затем выгрузить:
  adb shell dumpsys notification > before.txt
  ```
  После обновления выполнить ещё раз и сравнить файлы.
