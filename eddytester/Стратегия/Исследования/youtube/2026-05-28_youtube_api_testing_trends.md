## Insayty dlya kontenta eddytester
### Chto berut iz etikh video
- **Формат погружения с нуля** (freeCodeCamp, SDET QA) — делают видео‑курсы, которые становятся вирусными за счёт полноты.
- **Короткие анимированные объяснения** (Exponent, ByteByteGo) — упаковка сложного в 4‑9 минут, высокий retention.
- **Русскоязычная ниша** — Ulbi TV, Artsiom Rusau, Merion Academy показывают, что есть огромный спрос на контент на русском с акцентом на тестирование.

### Chego ne khvataet
- **Конкретные баги и кейсы из реальной работы** — в топе только общие учебные примеры, нет разборов проблем (например, «Почему не проходит авторизация», «Как искать баги в ответах 500»).
- **Автоматизация API (Rest Assured, Playwright, k6)** — почти не представлена (только Python в Caleb Curry), а это ключевая компетенция современного QA.
- **Продвинутые темы** — GraphQL‑тестирование, gRPC, WebSocket, работа с моками (WireMock, MockServer), нагрузочное тестирование (k6, Locust).
- **Практические чек‑листы и шаблоны** — нет видео «Как составить тест‑кейсы для API», «Чек‑лист тестирования REST».

### Kakie temy mozhno sdelat glubzhe
- **REST vs GraphQL vs gRPC: когда что использовать** — вместо общего обзора (Ulbi TV) сделать серию с конкретными тестовыми сценариями.
- **Postman: продвинутые фичи** — collection runner, data‑driven testing, monit, pre‑request scripts (в туториалах только basics).
- **API‑тестирование для микросервисов** — как тестировать взаимодействие между сервисами, контрактное тестирование (Pact).
- **Безопасность API** — OWASP API Top 10, тестирование rate limiting, авторизация на практике.

### Kakie tegi ispolzovat v svoikh postakh
- Основные: `api`, `postman`, `тестировщик`, `rest api`, `тестирование`, `qa`, `тестирование api` — обязательные.
- Для привлечения начинающих: `что такое api`, `api простыми словами`, `тестирование с нуля`.
- Для продвинутых: `автоматизация api`, `rest assured`, `graphql`, `k6`, `микросервисы`.
- Для ретаргетинга: `ошибки api`, `баги api`, `чек-лист api`.

### Unikalnoe, chto mozhet dat eddytester
- **Документированная экспертиза «из окопа»** — никакой зубрежки, только реальные кейсы: «Как я нашёл баг в REST‑методе PUT», «Почему Postman показал 200, а тесты упали».
- **Короткие видео (3‑5 минут) с конкретным решением проблемы** — в стиле Exponent, но сфокусированные на API‑тестировании.
- **Русскоязычные чек‑листы и шаблоны** в PDF к каждому видео — то, чего нет у конкурентов (Ulbi TV, Artsiom Rusau дают только лекции).
- **Серия «API‑тестирование за 15 минут»** — быстрый старт для тех, кто переходит с ручного тестирования на автоматизацию.
- **Разбор реальных API** (открытые: GitHub, Telegram, Stripe) — показать, как тестировать продукт “как в работе”.
