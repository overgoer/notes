## 1. СТАВКИ (BETS) — ПЕРЕСМОТР С РЕШЕНИЯМИ

### Bet #1: Practicum — единственный P0
**Вердикт:** 🟢 ОСТАВЛЯЕМ. НИЧЕГО НЕ МЕНЯЕТСЯ.

**Аргументация:** 45 человек сказали «хочу» — 0 заплатили. Free Trial сломан. Платёжки нет. Продукт есть — продаж нет. Ситуация идентична Day1.

**НО добавляем жёсткое условие:**
- Free Trial починен = условие запуска любых анонсов. Сломанный триал = конверсия в ноль.
- Платёжка — не «потом» и не «в процессе». Это блокер. Без неё Practicum не существует как продукт.

**Метрика:** Выручка Practicum. Цель: первые 10 000₽ до 12 июня.

**Риск:** Первый платящий用户 получит сломанный опыт. Митигация: протестировать весь флоу самому до анонса. Триал → регистрация → оплата → доступ → задачи.

**Действия (до 22 мая):**
1. Починить Free Trial API (Postgres auth, /free/api/users 404)
2. Подключить платёжку (Продамус / ЮMoney / любая)
3. Токен из хардкода → .env
4. Синхронизировать "15+ багов" — в боте, в лендинге, в посте одно число
5. Мост видео→бот: переписать 1 задачу под Candidates API

### Bet #2: Контент = трибуна для Practicum
**Вердикт:** 🟡 МЕНЯЕМ ПРОПОРЦИИ И ФИЛЬТР.

**Что изменилось с Day1:**
- 44 поста готовы (было 0)
- Датасет 219 постов проанализирован
- SCOUT подтвердил: для 1.5K канала 3-4 поста/нед — оптимум
- Топ-5 по вовлечению — дискуссионные посты и анонсы Practicum

**Новая формула: 50/30/20**
- 50% — вовлечение (механики, угадайки, вопросы, холивары)
- 30% — обучение (шпаргалки, баг-репорты, разборы)
- 20% — продажа (прямые CTA на Practicum)

**Почему:** Данные гайда по вовлечению показали — вовлечение даёт переходы и комментарии. Чистое обучение без вовлечения = 0 комментов. Продажа без прогрева = игнор.

**Фильтр для каждого поста (3 вопроса — все три должны быть "да"):**
1. Этот пост заставит остановиться и прочитать?
2. Этот пост вызовет реакцию (коммент/лайк/переход)?
3. Этот пост ведёт к Practicum (прямо или косвенно)?

**Действия:**
- Переписать сериал с банка на e-commerce (до Пт 22.05)
- Все 44 поста проверять через фильтр 3 вопросов перед публикацией
- Не публиковать пост, если хотя бы на один вопрос ответ "нет"

### Bet #3: Вовлечение через механику
**Вердикт:** 🟡 ЖЁСТКО ДОКРУЧИВАЕМ.

**Что показали данные:**
- 56% постов с комм — выше среднего по TG (норма 20-40%)
- 44% без комм — каждый второй пост мёртвый
- Топ по вовлечению: анонсы Practicum (45, 25, 17 комм), дискуссии (24, 15)
- Чистые гайды — 0 комментов. Мемы — лотерея.

**Новое правило (жёсткое):** Каждый пост обязан содержать минимум ОДНО из:
1. Конкретный вопрос в конце (не «что думаете?», а «GET может иметь тело? Да/Нет и почему»)
2. Механику (угадай баг, исправь меня, да/нет, две правды одна ложь)
3. Призыв поделиться опытом с низким порогом («➕ если было», «👇 кто сталкивался»)
4. Прямой CTA с низким порогом

**Убиваем форматы без вовлечения:**
- ❌ Чистые новости без комментария Эда
- ❌ Ссылки на статьи без своего текста
- ❌ Мемы без вопроса или вовлекающего элемента (кроме редких случаев)
- ❌ Посты без вопроса к аудитории

**Метрика:** Доля постов с >5 комм. Цель: >50% (сейчас ~30%).

### Bet #4: Контент-система работает
**Вердикт:** 🔴 КАРДИНАЛЬНАЯ СМЕНА ПОДХОДА.

**Проблема (данные):**
- Ресерчер выдал 17 исследованных тем
- 44 поста готовы к публикации
- На канале — 1-2 поста в неделю
- Система есть — публикаций нет

**Решение — «От коллекционирования к публикации»:**

1. **Понедельник = День Планирования.** Каждый понедельник: выбираешь 3 поста из 44, ставишь в календарь на неделю. Не редактировать. Не улучшать. Не переписывать. Публиковать.

2. **3 поста в неделю — жёсткая норма.** Не 5 (для 1.5K 5+ убьют retention). Не 1 (как сейчас — убивает momentum).

3. **Ресерчер — только для горячих тем.** 1 пост в неделю из ресерчера (баг в Twitter, новое в Postman, тренд). Остальные 3 — из готовых 44.

4. **Новый подход к «готово»:** Пост готов = пост публикуется. Не «напишу, когда будет идеально», а «напишу сейчас, исправлю по реакции».

**Метрика:** Опубликовано постов в неделю. Цель: 3-4. Факт (последние недели): ~1-2.
