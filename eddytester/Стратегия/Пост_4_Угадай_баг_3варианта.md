# Три ответа API. Два норм. Один — баг. Найди 🔥

Достал я значит купон на 100%, заказываю товар.

А API мне выдаёт три разных ответа. И я такой: «Ребят, а вы из одного сервиса или где?»

**Вариант A**
Сумма = 0. Купон сработал. Заказ создан (201).
Товар есть. Денег нет. Всё чётко.

**Вариант B**
Товар удалили из каталога. Я отправляю заказ — 201 Created.
Товара нет. Отгружать нечего. Но API рад.

**Вариант C**
Заказ создался. HTTP-статус — 200.
Заголовок Location? А нету.

---

Какой вариант — баг? Или все норм?

👇 A — сумма 0, бизнес в минус
👇 B — заказ мёртвого товара
👇 C — 200 вместо 201, RFC в пролёте

🔥 если видел такое в реальной работе
👍 если хоть раз получал 200 на POST и думал «ну норм же»

---

**Разбор (для тех, кто не любит гадать):**

Спойлер: баги — все три. Это не игра «найди один правильный», это реальность.

Но самый опасный — C.
200 вместо 201 — платёжный шлюз думает «ой, запрос не прошёл, давай ещё раз». Двойной платёж. Возвраты. Клиент орёт. Поддержка горит. Ты виноват.

A) Сумма 0 — бизнес-логика не проверяет минималку. Надо 422.
B) Мёртвый товар — сервер не проверил актуальность. Тоже 422.
C) 200 вместо 201 — RFC нарушен. POST создаёт ресурс → должен быть 201 + Location.

И да, я серьёзно. В одном реальном API я получил **все три ответа на один запрос**.
Просто потому что у них балансировщик ронял запросы на разные инстансы с разной версией кода.
