**Сериал 1/10: Мне дали API ecommerce стартапа**

Знакомые попросили потестировать API их стартапа. Документация - одна страница в Confluence. Три эндпоинта. Без примеров.

Коллега: "Там всё просто. GET /products, POST /orders, DELETE /cart. Дерзай".

Открываю Postman. GET /products. Жму Send.

200 OK. 10 товаров. Работает.

Дальше - граничные:

GET /products?page=2 -> 10 товаров. Ок.

GET /products?page=100 -> 200 OK. 0 товаров.

GET /products?page=-1 -> 200 OK. Те же 10 товаров.

GET /products?page=abc -> 500 Internal Server Error.

**Спойлер:** за неделю я нашёл 23 бага. Но первые 3 - за 5 минут.

Почему так вышло? В коде был обычный `LIMIT 10 OFFSET (page-1)*10`. OFFSET не знает, сколько всего записей - поэтому page=100 отдаёт пустой массив вместо 404. На page=-1 формула даёт OFFSET -10, что равно OFFSET 0 - сброс на первую страницу. А page=abc просто не проходит в integer.

Три бага:
1. Отрицательный page - молча сбрасывается на первую страницу
2. Несуществующая страница - пустой массив вместо 404
3. Невалидный page - сервер падает

**Итог:** 3 бага за 5 минут. Неплохое начало.

**Вопрос к вам:** А вы проверяете граничные значения page на новом API? Или доверяете документации? 👇
