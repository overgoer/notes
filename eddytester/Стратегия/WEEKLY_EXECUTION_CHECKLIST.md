# Weekly Execution Checklist

> **Цель:** Стратегия не лежит мёртвым грузом. Каждую неделю — 3 поста, замер метрик, корректировка.

---

## ПН — День планирования (30 мин)

### Выполнить до 12:00:
- [ ] Открыть Календарь.md
- [ ] Выбрать 3 поста на неделю из 44 готовых (максимум 1 из ресерчера)
- [ ] Записать номера постов в календарь
- [ ] Проверить: все ли 3 поста прошли «Фильтр 3 вопросов»?
- [ ] Если пост требует переписывания (сериал) — поставить дедлайн за 1 день до публикации
- [ ] Если Free Trial сломан — заменить продающий CTA на вовлекающий
- [ ] **ОТПРАВИТЬ ЭДДИ ПИСЬМО С ПЛАНОМ НА НЕДЕЛЮ**

---

## Каждый день перед постом (10 мин)

### Перед публикацией:
- [ ] Пост готов? (не «напишу вечером»)
- [ ] Фильтр 3 вопросов пройден?
- [ ] Ссылка на бота/лендинг работает? (открыть на телефоне)
- [ ] CTA есть и соответствует типу поста (вовлечение/продажа)
- [ ] Время публикации: 11:30 (короткий) или 18:30 (основной)
- [ ] **Опубликовать. Без «улучшений».**
- [ ] После публикации: записать в календарь статус «опубликован»

---

## СБ — Проверка прогресса (15 мин)

- [ ] Сколько постов вышло на этой неделе? (норма: 3)
- [ ] Если <3 — почему? Что помешало?
- [ ] Какие посты зарезервированы на следующую неделю?
- [ ] Есть ли проблемы с Free Trial / ботом / платёжкой?

---

## ВС — Ретроспектива (30 мин)

### Заполнить статистику в календаре:
- [ ] Просмотры (среднее за неделю)
- [ ] Комментарии (среднее, % постов с >5 комм)
- [ ] Переходы в бот (если есть UTM/статистика)
- [ ] Выручка Practicum
- [ ] Подписчики TG (прирост за неделю)

### Анализ:
- [ ] Какой пост дал максимум вовлечения? Почему?
- [ ] Какой пост провалился? Почему?
- [ ] Что изменить на следующей неделе?
- [ ] Запустить SCOUT по 1 новой теме конкурентов
- [ ] Запустить ресерчер по 1 новой теме (из topics.json pool)

### Решение по Practicum:
- [ ] Free Trial работает? Да/Нет
- [ ] Платёжка принимает деньги? Да/Нет
- [ ] Первые продажи есть?
- [ ] Если ДА на все 3 → **запускаем прямые продажи через каждый пост**

---

## Еженедельные задачи SCOUT

Каждое ВС Биззи запускает:

1. **run_researcher** — 1 новая тема из topics.json pool (не из researched)
2. **run_researcher** — 1 тема по конкурентам (по очереди: Русов → Лебедев → Дудник → снова)
3. **Проверка:** обновились ли конкуренты? Новые форматы? Новые продукты?

---

## Что делать, если...

| Ситуация | Действие |
|----------|----------|
| Пост не готов к дедлайну | Заменить на резервный из календаря. Не пропускать день. |
| Free Trial сломан | Все CTA на Practicum → вовлекающие CTA. Не продавать в битую воронку. |
| Нет идей на неделю | Открыть 44 готовых. Взять любые 3. Не думать, публиковать. |
| Провальная неделя (<2 постов) | Следующая неделя — 4 поста (догнать). |
| Хочется сделать «идеально» | Опубликовать как есть. Идеальный пост = опубликованный пост. |
| Ресерчер выдал конфликтные данные | Приоритет — данным из канала конкурента (просмотры/комм). Ресерчер — гипотеза. |
