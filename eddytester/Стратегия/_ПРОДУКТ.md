## 1. Free Trial API (бесплатный вход)

> Демо-версия, чтобы пощупать продукт перед покупкой.

**Репозиторий:** github.com/overgoer/free-trial-api
**Стек:** Node.js + Express + PostgreSQL
**Статус:** сломан (404, Postgres auth issues)

**Что делает:**
- Выдаёт trial API-ключи на 24 часа (POST /free/api/keys)
- Позволяет создавать/читать пользователей (CRUD lite)
- Тоже содержит баги (но меньше, чем полная версия)
- В каждом ответе поле _upsell → ссылка на полную версию

**Роль в воронке:** подсечка теплых подписчиков. Человек пробует бесплатно,
находит баги, ему нравится → покупает полную версию Practicum.

**Кому подходит:** всем. Даже тот, кто не заплатит — найдёт баги,
обсудит в комментах, создаст шум.

## 2. Practicum (v0-test-api, полная версия)

> Учебный API-симулятор с преднамеренными багами. Сам продукт.

**Репозиторий:** github.com/overgoer/v0-test-api
**Стек:** Node.js + Express + PostgreSQL
**Production:** http://85.193.81.51:8080

**Что делает:**
- REST API для управления пользователями (полный CRUD)
- Две версии: v1 (с багами) и v2 (исправленная)
- Swagger документация
- Postman коллекция + Jest/Supertest тесты
- Больше багов, сложнее сценарии, серьёзнее челлендж

**API v1 (баги):**
- GET /v1/api/users — баг: case-sensitive фильтр
- GET /v1/api/users/:id — баг: id-1 вместо id
- POST /v1/api/users — баги: 17 лет проходит, name с цифрами, age строка
- PATCH /v1/api/users/:id — баги: нет id в ответе, age строка, -age → 500
- DELETE /v1/api/users/:id — баги: не удаляет, невалидный id → 500

**v2:** все баги исправлены.

## Целевая аудитория (уточнено)

**Основная:**
- **Junior QA** и те, кто хочет войти в тестирование
- Не хватает практики API-тестирования для трудоустройства
- Хотят показать на собеседовании: «я тестировал реальный API, находил баги, писал автотесты»
- Готовы платить за практический опыт, который даёт преимущество

**Вторичная:**
- Middle QA, которые хотят прокачать API-тестирование
- Студенты курсов (Яндекс.Практикум, OTUS, Stepik), которым нужна практика

**Ключевой инсайт:** продукта такого нет на рынке. Есть ReqRes / JSONPlaceholder
(просто демо без багов), есть курсы (теория без практики).
Practicum — единственный, где можно реально искать баги и писать тесты.

## Конкурентное преимущество

- Реальные баги (не выдуманные учебные примеры)
- Две версии: баги → исправления (можно сравнить)
- Бесплатный вход через trial
- Дешевле курсов, конкретнее теории
- Можно сразу показать результат на собеседовании
