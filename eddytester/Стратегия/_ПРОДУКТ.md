# Practicum — v0-test-api

> Учебный API-симулятор с преднамеренными багами.
> Наш продукт. Все стратегические советы должны учитывать его.

---

## Что это

Node.js/Express сервер с REST API для управления пользователями.
Две версии: **v1 (с багами)** и **v2 (исправленная)**.
Учебный инструмент: QA-инженеры ищут баги в v1, сверяют с v2.

## Для кого

**Целевая аудитория:** QA-инженеры (middle), которые хотят:
- Научиться тестировать REST API
- Понять, как выглядят реальные баги в API (валидация, логика, безопасность)
- Перейти от ручного тестирования к автотестам (Postman → Newman → Jest/Supertest)
- Прокачать навык документирования багов
- Подготовиться к собеседованию (API-тестирование — частая тема)

## Пользовательский путь

1. QA-инженер узнаёт о Practicum (через Telegram-канал @eddytester)
2. Получает API-ключ (через бота или форму)
3. Тестирует API v1: находит баги, документирует
4. Сверяет с v2: какие баги исправлены
5. Пишет автотесты (Postman/Jest)
6. Получает фидбек, сравнивает свои находки с эталонным списком

## API Endpoints

**v1 (с багами):**
- GET /v1/api/users — баг: case-sensitive фильтр статуса
- GET /v1/api/users/:id — баг: использует `id-1` вместо `id`
- POST /v1/api/users — баги: age=17 проходит, name с цифрами, age строка
- PATCH /v1/api/users/:id — баги: нет id в ответе, age строка, отриц. age → 500
- DELETE /v1/api/users/:id — баги: не удаляет, невалидный id → 500

**v2:** все баги исправлены.

## Почему продаётся

QA-инженерам не хватает практики на реальных проектах.
Курсы дают теорию, но не показывают реальные баги в API.
Practicum — песочница, где можно безопасно искать баги, писать тесты,
получать опыт, который работодатели ждут на собеседованиях.

## Конкуренты

- ReqRes.in, JSONPlaceholder — нет багов, только демо
- ArtOfTesting, ToolsQA — теория без практики
- Stepik, OTUS — дорого, долго, не API-focused

Practicum выигрывает: реальные баги, две версии, сразу начать, дешевле.
