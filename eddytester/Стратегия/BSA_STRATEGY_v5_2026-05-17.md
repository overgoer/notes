### Bet #2: Контент = трибуна для Practicum
**Вердикт:** 🟡 МЕНЯЕМ ПРОПОРЦИИ. 50/30/20 — новая формула.

**Почему:** Данные SCOUT + Гайд по вовлечению:
- Топ-5 постов @eddytester — дискуссии и анонсы Practicum
- Чистые гайды = 0 комментов
- Вовлечение даёт переходы и комментарии

**Новая формула:**
- 50% вовлечение (механики, угадайки, холивары, вопросы)
- 30% обучение (шпаргалки, баг-репорты, разборы)
- 20% продажа (прямые CTA на Practicum)

**Фильтр 3 вопросов для каждого поста (ДО публикации):**
1. Заставит остановиться и прочитать?
2. Вызовет реакцию (коммент/лайк/переход)?
3. Ведёт к Practicum (прямо или косвенно)?

Если хотя бы один ответ «нет» — пост не выходит.

### Bet #3: Вовлечение через механику
**Вердикт:** 🟡 ДОКРУЧИВАЕМ ДО ЖЁСТКОЙ НОРМЫ.

**Данные:**
- 44% постов без комментов — каждый второй мёртвый
- Норма TG: 20-40% с комм. У Эдди: 56%. Выше среднего, но не предел.

**Новое правило (жёсткое):** Каждый пост содержит минимум ОДНО из:
1. Конкретный вопрос в конце (не «что думаете?», а «GET может иметь тело? Да/Нет и почему»)
2. Механику (угадай баг, исправь меня, да/нет, две правды одна ложь)
3. Призыв с низким порогом («➕ если было», «👇 кто сталкивался»)
4. Прямой CTA с низким порогом

**Убиваем форматы без вовлечения:**
- ❌ Чистые новости без комментария
- ❌ Ссылки на статьи без своего текста
- ❌ Мемы без вопроса
- ❌ Посты без вопроса к аудитории

**Метрика:** >50% постов с >5 комм (сейчас ~30%).

**Что отменяем из v4:** Запрет на посты из пула ресерчера → разрешаем 1 в неделю.

## 3. КАЛЕНДАРЬ — СТРУКТУРА

**Файл:** `/root/obsidian-vault/eddytester/Стратегия/Календарь.md` (пока write_file ограничен Стратегией/)
**Когда сможем писать в Контент/:** перенесём.

**Формат записи:**
```
| Дата | Время | Тип | Формат | Тема | CTA | Статус | Ссылка |
```

- **Дата:** ДД.ММ (18.05)
- **Время:** 11:30 или 18:30 МСК
- **Тип:** вовлечение / обучение / продажа
- **Формат:** № из 10
- **Тема:** кратко
- **CTA:** что делаем
- **Статус:** запланирован / написан / опубликован
- **Ссылка:** на пост (после публикации)

**Принципы:**
1. Пн, Ср, Пт — 3 поста
2. 18:30 — основные, 11:30 — короткие (шпаргалки)
3. 2 вовлечения + 1 обучение ИЛИ 1 продажа + 1 вовлечение + 1 обучение
4. Каждый пост = номер из 44 готовых (искл: горячая тема 1/нед)
5. Вс — ретроспектива

## 9. НЕПОБЕДИМОСТЬ — ЧТО ЭТО ЗНАЧИТ

**Непобедимость @eddytester для начинающего QA — это когда:**

1. **Practicum — единственный API-симулятор с реальными багами.** Ни у кого больше нет живого API, который можно сломать. Русов учит пользоваться Postman. Ты даёшь API и говоришь «найди баги».

2. **Каждый пост — кирпич в доверии.** После 10 постов подряд с механиками, баг-репортами, шпаргалками — подписчик знает: «Этот парень шарит. Я куплю его Practicum».

3. **3 поста в неделю — ритм, который не ломается.** Не 7 (выгоришь), не 1 (тебя забудут). 3 — оптимум для 1.5K.

4. **Ты не гонишься за конкурентами.** Ты играешь между их слотами. Ты не копируешь их форматы. Ты делаешь то, чего нет у них — живой API с багами.

5. **Сегодня > идеально.** 44 поста готовы. Free Trial чинится. Платёжка подключается. Не ждать. Публиковать. Продавать.
