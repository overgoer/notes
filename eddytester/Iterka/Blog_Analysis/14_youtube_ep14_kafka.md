# Выпуск #14 — Андрей Серебрянский: Как работает Kafka и зачем она нужна

**Ссылка:** https://www.youtube.com/watch?v=6F0Zt631ipM
**Канал:** Iterka (Эдгар Бурёнкин)
**Гость:** Андрей Серебрянский — продакт-оуэр стриминговой платформы в Райффайзенбанке, выступал на Kafka Summit London и J Point.

## Ключевые темы

| Тема | Описание |
|---|---|
| **Путь в IT** | Андрей с нуля выучился программировать за ~6 месяцев. Окончил ВШЭ по менеджменту, самоучка. Первый коммерческий опыт — стартап за 10 000 руб. Нашёл первую работу через месяц после начала обучения. |
| **Что такое стриминг данных** | Альтернатива батч-процессингу (когда данные передаются «пачками» раз в час/день). Стриминг — передача каждого события мгновенно, в реальном времени. Решает проблему задержек: пользователь видит данные секундной давности, а не вчерашние. |
| **Apache Kafka** | Стандарт индустрии для стриминга. Брокер сообщений, который принимает события, сохраняет их (даже при падении) и рассылает всем подписчикам. Работает через топики (topics) — логические группы сообщений. |
| **CDC (Change Data Capture)** | Как данные попадают в Kafka из базы: используется Write-Ahead Log (WAL) — база сначала пишет изменения в журнал, затем накатывает на себя. Стриминговая платформа читает этот лог и публикует изменения в Kafka. |
| **Batch Processing vs Streaming** | Батч: дёшево, надёжно (знаем, что все данные за период пришли), но с задержкой. Стриминг: быстро, но сложнее (нет гарантии конца потока, возможны дубли, нарушение порядка). |
| **Проблемы распределённых систем** | Дублирование сообщений, гарантия порядка (клиент положил 500 руб, потом купил обед — порядок критичен). Нет серебряной пули — нужно проектировать код с учётом этих сценариев. |
| **Scalability** | Горизонтальное масштабирование — Kafka легко добавляет ноды. Рост с 3 до 30 серверов может покрыть рост с 10K до 10М сообщений/сек. Контраст с вертикальным масштабированием (ограничено железом). |
| **Event Streaming в real life** | Фрод-мониторинг (транзакция в Занзибаре в 4 утра, если днём был в Москве), визуализация загрузки саппорта на телевизорах, конвертация валют по актуальному курсу, интеграция с ChatGPT для контекстного ответа клиенту. |
| **Confluent** | Компания-лидер (выходцы из LinkedIn, создатели Kafka Open Source). Проводят Kafka Summit. Выпускают много обучающего контента, курсы, статьи (включая интеграцию с ChatGPT). |
| **Open Source & импортозамещение** | Kafka, Flink, Kafka Streams — полностью Open Source на Java. Уход западных вендоров не влияет — всё поднимается на Linux-серверах. Нужны только квалифицированные специалисты. |
| **Состояние индустрии в России** | Централизованные стриминговые платформы только начинают появляться (Тинькофф, Райффайзен, Яндекс, Сбер). В мире тема активно развивается — Kafka Summit London собрал 2000+ участников. |
| **Обучение и вход в специальность** | Книга «Designing Data-Intensive Applications» Мартина Клеппмана (библия дата-инженера). Курсы Confluent (бесплатные, качественные). Нужна хорошая Java, интерес к распределённым системам. |

## Наблюдения и инсайты

1. **Нетипичный путь в IT** — менеджмент ВШЭ, самообучение за полгода (не через онлайн-школы, а через подбор курсов из программы Computer Science ВШЭ и альтернатив на YouTube/зарубежных платформах). Первый проект — за 10 000 руб за 3 месяца.

2. **Точка входа в стриминг** — стартап с голосовалкой на 200 человек — неожиданный первый опыт с real-time интеграциями, который в итоге привёл в Тинькофф, а затем в Райффайзенбанк.

3. **Kafka как «склеивающий слой»** — стриминговая платформа не просто передаёт данные, а снимает dependency между командами. Любая команда может подписаться на нужные события и получать их без согласования с владельцем данных.

4. **Duality разработки** — разработчики стриминговой платформы сталкиваются с проблемами памяти (out of memory, утечки) гораздо чаще, чем обычные бэкенд-разработчики. Доклад Андрея на Kafka Summit был именно про это.

5. **Фрод-детекция как killer feature** — определение мошеннических транзакций в реальном времени — ключевой бизнес-кейс для event streaming в банках.

6. **Разрыв между миром и Россией** — на Западе стриминг доступен как сервис из коробки (Confluent, Google, AWS) для любых компаний. В России — только крупные банки и корпорации строят свои платформы. Рынок только зарождается.

7. **Open Source = устойчивость** — полная независимость от импортных лицензий (Kafka, Flink, Java — всё Open Source).

8. **Книга Клеппмана как must-read** — «Designing Data-Intensive Applications» упоминается как библия для всех, кто хочет понять распределённые системы, брокеры сообщений и базы данных.

9. **Конференции как драйвер развития** — Kafka Summit London, J Point, подача докладов — нетривиальный, но эффективный путь для нетворкинга и профессионального роста.

10. **Совет новичкам** — искать команды, которые уже используют Kafka, пусть даже не в стриминге. Опыт работы с Kafka — отличный входной билет в индустрию event streaming.
