**Гипотеза:** Основная аудитория — manual QA 1-2 года. Уже работает, понимает ценность практики, готов платить.

## 2.2 Ценность Practicum для клиента

Гипотезы, от поверхности к сути:

**Гипотеза 1: Песочница** — "API, который можно тестировать без страха сломать прод"
→ Слабо: "песочница" не цепляет эмоционально

**Гипотеза 2: Охота за багами** — "API с 30 багами — найди их все"
→ Сильнее: азарт, геймификация, дофамин от найденного бага

**Гипотеза 3: Прокачка для работы** — "Найдешь баги у нас — найдешь на работе за час"
→ Утилитарно, нет эмоции

**Гипотеза 4: Собеседование (ставка)** — "Покажи на собеседовании: я тестировал реальный API и нашел 30 багов"
→ **Герой продукта:** не ученик, а охотник за багами. Доказательство навыка вместо "я смотрел курсы".

**Гипотеза 5 (составная):**
> "Хватит смотреть — начинай тестировать. 30 багов ждут."

Три уровня ценности:
1. **Практика** — единственный способ научиться
2. **Азарт** — найти баг интереснее чем смотреть лекцию
3. **Портфолио** — доказательство навыка для работы

Первые 5 секунд видео:
> "Ты смотришь 50-й туториал по Postman, а в реальном API все равно не знаешь где искать баги. Я дам тебе API с 30 багами. Найди их."

## 2.3 Practicum как продукт

**Модель монетизации:**

1. **Бесплатно** — демо-API (5-7 багов), YouTube-контент, чек-листы
2. **Practicum Base (~1000 руб/мес)** — 30+ багов, v1+v2, Swagger, Postman коллекция
3. **Practicum PRO (~3000 руб/мес)** — SQL-доступ, ротация багов ежемесячно, сертификат
4. **Enterprise (договорная)** — кастомные баги, доступ для команды

**Practicum = HackTheBox for API Testing.** Продукта такого нет. YouTube — воронка в него. Тренажер даёт контент для видео, видео даёт трафик тренажёру.

# Часть 3. Контент-стратегия

## 3.1 Формат: сериал "API-баг недели"

Один баг за выпуск. 5-8 минут. Без воды.

**Структура выпуска:**

1. **Ситуация** — "Есть POST /users, принимает name и age"
2. **Пауза 5 сек** — "что здесь может сломаться?"
3. **Воспроизведение** — curl, показываем баг
4. **Вскрытие** — почему произошло (SQL, JSON, типы, фреймворк, сеть)
5. **Чек-лист** — 3-5 пунктов для этого типа багов (PDF в описании)
6. **CTA** — ссылка на демо-API / Practicum

**Почему сработает:**
- Ниша пустая — мы первые
- Сериальность — подписки, retention, алгоритм
- Узнаваемые слова в заголовке (валидация, авторизация)
- Баг в заголовке — кликбейт без обмана
- Пауза с вопросом — вовлечение через комментарии

## 3.2 Формула заголовка

```
[Концепт] + [Баг] + [Контекст]
```

**Концепт:** валидация, авторизация, пагинация, идемпотентность, кэширование, rate limiting, status code

**Баг:** "сломан", "пропустил", "не работает", "создает дубликат"

**Контекст:** POST /users, DELETE /:id, Postman, REST API, SQL

### Матрица заголовков

| Концепт | Баг | Контекст | Готовый заголовок |
|---------|-----|----------|-------------------|
| Валидация | пропустил age=-5 | POST /users | Валидация API сломана: POST /users пропустил age=-5 |
| Авторизация | прошел без токена | PATCH /:id | Авторизация API: почему запрос без токена прошел |
| Пагинация | объект в двух страницах | GET /users | Пагинация API: тот же объект на двух страницах |
| Идемпотентность | двойной заказ | PUT /orders | PUT создает дубликат — ты проверял идемпотентность? |
| Кэширование | чужие данные | Cache-Control | Твой API кэширует чужие данные. Вот как проверить |
| Status Code | 500 вместо 400 | невалидный JSON | 500 вместо 400 — баг в error handling API |
| Rate Limiting | 1000 запросов прошло | POST /api | Rate Limiting не работает: 1000 запросов прошли |
| DELETE | не удаляет | DELETE /users/:id | DELETE который не удаляет — баг которому 2 года |
| Content-Type | text/plain вместо JSON | GET /users | Content-Type: text/plain — API не возвращает JSON |
| Side effects | 2 объекта вместо 1 | POST /users | POST создает два объекта. Один лишний |
| Фильтрация | регистр ломает | GET ?status= | Status filter: MINOR не равно minor |
| Error handling | stack trace в ответе | любой эндпоинт | В ответе API — stack trace. Так нельзя |

### Топ-3 для старта

1. **Валидация API сломана: POST /users пропустил age=-5** — понятно, страшно, конкретно
2. **Авторизация API: почему запрос без токена прошел (и это баг)** — безопасность, страх
3. **PUT создает дубликат — ты проверял идемпотентность?** — проф. стыд, провокация

### Правила заголовков

1. Первые 3 слова — самые важные (обрезаются в рекомендациях)
2. Концепт — в начало
3. Длина — 40-60 символов
4. Никаких "тестирование API для начинающих" — сольют с туториалами
5. Конкретика: "age=-5" лучше чем "невалидные данные"
6. Вопрос в конце повышает CTR

## 3.3 Сезон 1: "Основы REST API — ищем баги"

10 выпусков. График: каждый четверг.

| № | Тема | Баг | Чек-лист |
|---|------|-----|----------|
| 1 | POST /users — age | age=17 проходит вместо 18 | Граничные значения |
| 2 | GET /users/:id | Ищет по id-1 вместо id | Параметры ID |
| 3 | DELETE /users/:id | SELECT вместо DELETE | CUD операции |
| 4 | PATCH /users/:id | id пропадает из ответа | Полнота PATCH |
| 5 | Status filter | Case-sensitive SQL | Фильтрация |
| 6 | POST side effects | 2 пропуска вместо 1 | Побочные эффекты |
| 7 | Content-Type | text/plain вместо JSON | Заголовки |

# Часть 4. Воронка и инфраструктура

## 4.1 Схема

```
YouTube видео
  → описание: ссылка на демо-API
  → демо-API: 5 багов, Swagger, CTA "30+ багов в Practicum"
  → Practicum: полный продукт
```

## 4.2 Демо-API (для съемок)

Отдельный инстанс на сервере. Баги НЕ пересекаются с Practicum.

- 5-7 багов, зрелищные и понятные
- Только API (без SQL-доступа)
- Swagger документация
- В каждом видео: "хочешь найти 30+ таких багов? ссылка"

## 4.3 Practicum (продукт)

- Демо-баги — статические (всегда доступны)
- Practicum-баги — ротация раз в месяц
- Студент Practicum получает то, чего нет в бесплате и в видео

## Нерешенные вопросы
