# Стратегия YouTube-канала: eddytester — неоспоримо лучший по API-тестированию

**Дата:** 2026-05-28
**На основе:** YouTube Data API исследование (66 видео) + аудит конкурентов

---

## Главная идея

Никто на русском YouTube не делает разбор **конкретных багов API**. Все — лекции, обзоры, туториалы. Формат "вот баг, вот как до него додуматься" отсутствует.

Наша ниша: **API-баг: вскрытие**. Один баг за выпуск. 5-8 минут. Без воды.

---

## Контент-формат: сериал "API-баг недели"

### Структура одного выпуска

1. **Ситуация** — "Есть POST /users, принимает name и age"
2. **Задание зрителю** — пауза 5 секунд: "что здесь может сломаться?"
3. **Воспроизведение** — curl, показываем баг
4. **Вскрытие** — почему баг произошел (SQL, JSON, типы, фреймворк, сеть)
5. **Чек-лист** — 3-5 пунктов для этого типа багов (раздаем PDF)
6. **Попробуй сам** — ссылка на демо-API / Practicum

### Почему это сработает

- Никто на русском ютубе не делает разбор багов
- Каждый выпуск — продажа Practicum без прямой рекламы
- Мы единственные у кого есть свое API для демонстрации багов
- Сериальность = подписки, retention, алгоритм
- Пауза с вопросом = вовлечение через комментарии

### Демо-API (для съемок)

Отдельный инстанс на том же сервере (другой порт или поддомен). Баги не пересекаются с Practicum.

- 5-7 багов, зрелищные и понятные
- Никакого SQL доступа, только API
- Swagger документация
- В конце каждого видео: "хочешь найти 30+ таких багов? ссылка"

### Practicum (продукт)

- В Practicum баги ротируются — раз в месяц свежие
- Студент Practicum получает то, чего нет в бесплатном доступе и в видео
- Демо-API — воронка, Practicum — монетизация

## Ключевые принципы

1. **Никаких лекций.** Только баги. Зритель пришел за мясом.

# Стратегия заголовков — как привлекать зрителей

**Дата:** 2026-05-28
**Метод:** Анализ YouTube API — 80 поисковых запросов по багам, 66 топ-видео в нише

---

## Рынок: что показал поиск

Поиск по всем ключевым словам (баг в валидации, авторизации, пагинации, POST, rate limiting, idempotency, caching, error handling) показал:

**Ноль релевантного контента.** Результаты:
- Баг-баунти / хакинг — про безопасность, не про тестирование
- Общие туториалы — что такое пагинация, как работает кэш
- Единственное близкое — "How to log a simple API bug" — но это про заполнение баг-трекера

**Ниша пустая.** Когда человек ищет "баг в валидации API" — он получает либо хакеров, либо абстрактные лекции. Конкретного разбора бага для тестировщика нет ни на русском, ни на английском.

---

## Формула заголовка

```
[Концепт] + [Баг] + [Контекст]
```

Где:

**Концепт** — узнаваемое слово из тестирования, которое ищут:
- Валидация, авторизация, аутентификация, пагинация
- Идемпотентность, кэширование, rate limiting
- Status code, error handling, безопасность
- Content-Type, заголовки, CORS

**Баг** — конкретная проблема, эмоция:
- "сломан", "пропустил", "не работает", "создает дубликат"
- "почему X, а должно быть Y"
- "как я нашел", "ты проверял?"

**Контекст** — привязка к инструменту или технологии:
- POST /users, DELETE /api, PATCH /:id
- Postman, curl, REST API
- SQL, JSON, HTTP

---

## Матрица заголовков

| Концепт | Баг | Контекст | Готовый заголовок |
|---------|-----|----------|-------------------|
| Валидация | пропустил age=-5 | POST /users | Валидация API сломана: POST /users пропустил age=-5 |
| Авторизация | прошел без токена | PATCH /:id | Авторизация API: почему запрос без токена прошел |
| Пагинация | объект в двух страницах | GET /users?page= | Пагинация API: тот же объект на двух страницах |
| Идемпотентность | двойной заказ | PUT /orders | PUT создает дубликат — ты проверял идемпотентность? |
| Кэширование | чужие данные | Cache-Control | Твой API кэширует чужие данные. Вот как проверить |
| Status Code | 500 вместо 400 | невалидный JSON | 500 вместо 400 — баг в error handling API |
| Rate Limiting | 1000 запросов прошло | POST /api | Rate Limiting не работает: 1000 запросов и все прошли |
| DELETE | не удаляет | DELETE /users/:id | DELETE который не удаляет — баг которому 2 года |
| Content-Type | text/plain вместо JSON | GET /users | Content-Type: text/plain — твой API не возвращает JSON |
| Side effects | 2 объекта вместо 1 | POST /users | POST создает два объекта. Один лишний |
| Фильтрация | регистр ломает фильтр | GET ?status= | Status filter: почему MINOR не равно minor |
| Error handling | stack trace в ответе | любой эндпоинт | В ответе API — stack trace. Так нельзя |

## Топ-3 для старта

| # | Заголовок | Почему |
|---|-----------|--------|
| 1 | **Валидация API сломана: POST /users пропустил age=-5** | Понятно, страшно, конкретно. Любой тестировщик сталкивался |
| 2 | **Авторизация API: почему запрос без токена прошел (и это баг)** | Безопасность, страх. Хайпово |
| 3 | **PUT создает дубликат — ты проверял идемпотентность?** | Профессиональный стыд, провокация. Вовлекает |

---

## Почему это сработает

1. **Ниша пустая** — по всем концептам нет контента. Мы первые
2. **Алгоритм любит сериалы** — "API-баг #1", "#2", "#3" — подписки, retention
3. **Узнаваемые слова** — валидация, авторизация, пагинация — люди это ищут
4. **Баг в заголовке** — кликбейт без обмана. В видео реально баг
5. **Сериальность** — зритель посмотрел #1, хочет #2. Подписка в один клик

---

## Правила заголовков

1. Первые 3 слова — самые важные (обрезаются в рекомендациях)
2. Концепт — в начало: "Валидация API..." а не "Как я нашел баг в валидации"
3. Длина — 40-60 символов
4. Никаких "тестирование API для начинающих" — это уже есть, нас сольют с туториалами
5. Конкретика: "age=-5" лучше чем "невалидные данные"
6. Вопрос в конце: "ты проверял?" — повышает CTR

## 3. Международные аналоги: HackTheBox и TryHackMe

Эти платформы — ближайшие аналоги по модели "тестируй в песочнице".

**HackTheBox** (кибербезопасность, 2M+ пользователей):
- **Free** — ограниченный доступ
- **VIP+** — $25/мес или $223/год ($18.6/мес)
- Отдельная подписка на Academy — $8-18/мес
- Pro Labs — $49/мес (AD-среды)

**TryHackMe** (кибербезопасность, вход):
- **Free** — ограниченные комнаты, 1 час/день VM
- **Premium** — $10.5/мес (годовая) или $16.99/мес
- 900+ комнат, learning paths, сертификаты

**Общая модель:**
- Бесплатно — ограниченный доступ (воронка)
- $10-25/мес — полный доступ
- Enterprise — корпоративные подписки

**Что это значит для Practicum:**
- Модель "бесплатно + платный доступ" валидирована рынком
- Цена $10-20/мес — рынок принимает
- Разница: HackTheBox/TryHackMe про безопасность, Practicum про тестирование API. Никто не делает то же самое для QA
- **Потенциально Practicum может занять нишу "HackTheBox for API Testing"**

## 5. Practicum — продукт или воронка?

**Международный опыт (HackTheBox/TryHackMe):**
- Продукт живёт сам по себе (подписки)
- Контент + комьюнити = удержание
- Enterprise = B2B доход

**Для Practicum:**
- **Продукт** — тренажер по API-багам. Люди платят за доступ к багам
- **Воронка** — YouTube → дарят трафик Practicum. Practicum → дарят темы для YouTube
- **Дополнительно:** консультации / ревью тестов для компаний (B2B)

**Рекомендуемая модель:**
1. **Бесплатно:** демо-API (5-7 багов), YouTube контент, чек-листы
2. **Practicum Base (~1000 руб/мес):** 30+ багов, v1+v2, Swagger, Postman коллекция
3. **Practicum PRO (~3000 руб/мес):** SQL-доступ, ротация багов ежемесячно, сертификат, поддержка
4. **Enterprise (договорная):** кастомные баги, доступ для команды, интеграция с курсами
