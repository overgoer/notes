## Инсайты для контента eddytester

### Чего не хватает в текущих видео

1. **Конкретные баги и кейсы из реальной работы** — в топе только учебные примеры. Нет разборов "Почему сервер вернул 500", "Как искать баги в ответах API".
2. **Автоматизация API** — Rest Assured, Playwright, k6 почти не представлены, а это ключевая компетенция современного QA.
3. **Продвинутые темы** — GraphQL-тестирование, gRPC, WebSocket, моки (WireMock, MockServer), нагрузка (k6, Locust).
4. **Практические чек-листы и шаблоны** — нет видео "Как составить тест-кейсы для API", "Чек-лист тестирования REST".

### Что уникального может дать eddytester

1. **Экспертиза "из окопа"** — реальные кейсы: "Как я нашёл баг в PUT", "Почему Postman показал 200, а тесты упали"
2. **Короткие видео (3-5 мин) с конкретной проблемой** — в стиле Exponent, но про API-тестирование
3. **Чек-листы и шаблоны** в PDF — то, чего нет у Ulbi TV и Artsiom Rusau
4. **Серия "API-тестирование за 15 минут"** — быстрый старт для ручных тестировщиков
5. **Разбор реальных API** — GitHub, Telegram, Stripe — "как в работе"

### Какие теги использовать в постах

- Основные: `api`, `postman`, `тестировщик`, `rest api`, `тестирование`, `qa`, `тестирование api`
- Для начинающих: `что такое api`, `api простыми словами`, `тестирование с нуля`
- Для продвинутых: `автоматизация api`, `rest assured`, `graphql`, `k6`, `микросервисы`
- Для вовлечения: `ошибки api`, `баги api`, `чек-лист api`
