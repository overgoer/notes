# Настройка NS серверов и API токена для pupupuu.com

## Текущий статус
✅ Домен `pupupuu.com` добавлен в Cloudflare
❌ NS серверы ещё не изменены у регистратора

## Шаг 1: Получение NS серверов от Cloudflare

1. Войдите в панель Cloudflare
2. Выберите домен `pupupuu.com`
3. На главной странице домена найдите раздел **"Nameservers"**
4. Cloudflare покажет 2 NS сервера, например:
   ```
   lara.ns.cloudflare.com
   marty.ns.cloudflare.com
   ```
5. **Запишите эти NS серверы**

## Шаг 2: Определение регистратора домена

Вам нужно узнать, где зарегистрирован домен `pupupuu.com`:

1. Выполните команду в терминале:
   ```bash
   whois pupupuu.com | grep -i "registrar"
   ```
   Или
   ```bash
   dig SOA pupupuu.com
   ```

2. Возможные варианты:
   - **Porkbun** (если регистрировали там)
   - **Cloudflare Registrar** (если покупали через Cloudflare)
   - Другой регистратор

## Шаг 3: Изменение NS серверов у регистратора

### Если регистратор Porkbun:
1. Войдите в аккаунт Porkbun
2. Перейдите к управлению доменом `pupupuu.com`
3. Найдите раздел **"Nameservers"** или **"DNS Management"**
4. Замените существующие NS серверы на:
   ```
   lara.ns.cloudflare.com
   marty.ns.cloudflare.com
   ```
5. Сохраните изменения

### Если другой регистратор:
- Найдите раздел управления DNS или Nameservers
- Замените NS серверы на полученные от Cloudflare
- Сохраните изменения

### Если Cloudflare Registrar:
- NS серверы уже настроены автоматически
- Переходите к шагу 4

## Шаг 4: Ожидание распространения DNS

Изменение NS серверов занимает:
- **1-48 часов** (обычно 1-2 часа)
- Проверка статуса:
  ```bash
  dig NS pupupuu.com
  ```
  Должны отображаться Cloudflare NS серверы

## Шаг 5: Создание A-записи в Cloudflare

**После распространения NS** (когда `dig NS pupupuu.com` показывает Cloudflare):

1. В панели Cloudflare выберите домен `pupupuu.com`
2. Перейдите в **DNS → Records**
3. Нажмите **"Add record"**
4. Настройте A-запись:
   - Type: `A`
   - Name: `@` (или оставьте пустым для корневого домена)
   - IPv4 address: `217.144.185.210`
   - Proxy status: **Оранжевое облако** (проксирование включено)
   - TTL: `Auto`
5. Сохраните

## Шаг 6: Создание API токена Cloudflare

Для автоматической настройки сервера:

1. В панели Cloudflare: **My Profile → API Tokens**
2. Нажмите **"Create Token"**
3. Выберите шаблон **"Edit zone DNS"**
4. В разделе **"Zone Resources"** выберите зону `pupupuu.com`
5. Нажмите **"Continue to summary"**
6. Нажмите **"Create Token"**
7. **Скопируйте и сохраните токен в безопасном месте!**

## Шаг 7: Проверка настройки

После настройки NS и A-записи:

1. Проверьте DNS разрешение:
   ```bash
   dig pupupuu.com
   ```
   Должен вернуть `217.144.185.210`

2. Проверьте проксирование Cloudflare:
   ```bash
   curl -I http://pupupuu.com/
   ```
   В заголовках должен быть `CF-Ray` (Cloudflare)

## Проблемы и решения

### NS серверы не меняются:
- Убедитесь, что домен не имеет "Domain Lock"
- Проверьте, что у вас есть права администратора
- Возможно, нужно подтвердить email от регистратора

### DNS не распространяется:
- Подождите до 48 часов
- Проверьте командой: `dig NS pupupuu.com @8.8.8.8`

### Cloudflare не показывает домен:
- Возможно, домен был удалён из Cloudflare
- Добавьте домен заново: **"Add a Site"**
