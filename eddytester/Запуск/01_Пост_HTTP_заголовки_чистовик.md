**🔥 Content-Type**

Content-Type в запросе - это униформа. Одел `application/json` - значит, "я JSON". Одел `text/plain` - значит, "я просто текст".

Большинство фреймворков вообще не проверяют униформу. 
И чем больше остается на откуп фреймворку - тем больше дыр в безопасности сервиса.
Схема, rate limit, логи: всё валится, уже писал подробно тут 👉  https://t.me/eddytester/417 и всем рекомендую заглянуть.

Проверяется за 10 секунд:
```
curl -X POST /api/users \
  -H "Content-Type: text/plain" \
  -d '{"role": "admin"}'
```

200 OK вместо 415 Unsupported Media Type = дверь открыта, звоним команде.

**⚡️ Cache-Control**

Самый коварный заголовок - баг плавающий.

Обновил данные - API отвечает старыми. Перезашёл - новыми. "Магия". На самом деле - сервер закэшировал ответ и радует пользователей вчерашними новостями.

Я такое ловил на проде: пользователи жалуются, что данные не обновляются. Девопсы говорят "всё чистили". А ответ всё равно летит из кэша, потому что Cache-Control не настроен.

Тоже 10 секунд на локализацию и все поймешь:
- POST на создание → GET на чтение.
  Ответ без `Cache-Control: no-cache` ? С вероятностью 90% там кэш.
- PUT/PATCH а потом снова GET. Свежие данные? Всё ок.
Закэшированное старьё? -> Баг репорт!
- Если баг «плавает» - кинь в URL параметр `?v=1`. Баг исчез? Поздравляю, ты локализовал проблему.

👉 Подробный разбор кэширования с примерами: https://t.me/eddytester/398

**🛡 CORS**

Любимая отмазка фронтендеров: "CORS - это не наша проблема".

На самом деле CORS - это ~~пояс верности~~ акт доверия.
Сервер говорит браузеру: "этому домену я доверяю".

Если CORS настроен через `*` - доверять можно всем. Включая сайт, который украдёт твои данные.

10 секунд на локализацию:
Как проверять: шлёшь запрос с `Origin: https://evil.com`.
Смотришь `Access-Control-Allow-Origin` в ответе. Если там `*` или твой `evil.com` - у тебя проблемы.
