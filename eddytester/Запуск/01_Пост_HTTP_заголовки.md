## Вариант А: "3 HTTP-заголовка, которые я проверяю первыми"

Заголовки — это как правила дорожного движения для API.
Можно их не знать и ездить как получится. Но когда случается авария — виноват будешь ты.

Я тестирую API больше 5 лет.
И есть 3 заголовка, которые я проверяю ещё до того, как смотрю тело ответа.

**Content-Type**

Казалось бы, база. 
Но я видел API, который на JSON-запросе с заголовком `text/plain` возвращал 200 OK с битым телом.
Потому что фреймворк пытался спарсить XML, не находил корневой элемент и отдавал пустой ответ.

Без ошибки. Без предупреждения. Просто пустота.

Проверка: отправляешь тот же запрос с разными Content-Type и смотришь, меняется ли поведение.

**Cache-Control**

Кэш — это когда API отвечает быстро. Даже если ответ уже неактуален.
Я находил баги, которые жили в кэше сутками — обновляешь данные, а API отдаёт старьё.

Проверка: делаешь POST на создание → GET на чтение — и смотришь Cache-Control в ответе.

**CORS**

Про CORS говорят "это фронтенд-проблема".
Нет. CORS — это как API говорит браузеру: "этому сайту я доверяю, остальным — нет".
Когда CORS настроен неправильно, твой API могут дёргать с любого сайта.

Проверка: отправляешь запрос с Origin: `https://evil.com` — и смотришь, что в ответе.

## Вариант Б: "Почему Content-Type — самый опасный заголовок"

(более узкий фокус, глубже)

Content-Type для новичка — просто техническое поле. Поставил `application/json` в Postman и забыл.

А теперь представь: сервер ждёт JSON.
Ты отправляешь всё правильно. Но Content-Type — `text/plain`.

Что сделает сервер?

Некоторые фреймворки спарсят тело как есть и вернут 200 OK.
Другие — упадут с 500.
Третьи — вернут 200 OK, но в теле будет `null`.

И ни один не скажет тебе: "эй, ты забыл указать тип данных".

Я такое находил в реальных проектах:
- Express-приложение, которое на `text/plain` отдавало 200 с пустым телом
- Spring Boot, который возвращал XML вместо JSON (потому что content negotiation по заголовку Accept)
- Flask, который игнорировал Content-Type и парсил как придётся

**Что делать:**
Дёрни каждый эндпоинт с Content-Type: `text/plain`, `application/xml`, `multipart/form-data`, `application/json` (правильный).

Если хотя бы один вариант не вернул 415 (Unsupported Media Type) — у тебя проблема.
