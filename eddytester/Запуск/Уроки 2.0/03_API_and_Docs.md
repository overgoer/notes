## 📦 HTTP <br>
HyperText Transfer Protocol

Это строгий "почтовый конверт". <br>
Ты не можешь просто крикнуть серверу. Ты должен запаковать команду в HTTP-запрос.

Конверт нужно заполнять правильно, иначе письмо не дойдет.

### 📦 HTTP <br>(HyperText Transfer Protocol)

**API** - Какие действия можно делать на нашем сервере <br>
**HTTP** - технический язык общения с сервером.<br><br><br>
**Документация** покрывает связку между бизнес логикой и технической частью.<br>

# 🧬 Запрос HTTP
| Часть | Что это? | Как было у нас |
|---|---|---|
| **Метод** | Какое действие? | `POST` (Создать) |
| **URL (Endpoint)** | Какую кнопку жмем? | `/v1/api/users` |
| **Body (Тело)** | Какие данные? | `{"name":"John1"}` |

# 🎯 Итог

1. Мы запаковали HTTP-запрос.
2. HTTP-курьер доставил его к API (пульту сервера).
3. API выполнило работу (с багом!) и вернуло HTTP-ответ с кодом `200`.
