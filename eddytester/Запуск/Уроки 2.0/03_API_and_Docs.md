  **Блок 1: Практика — Первый взлом (Postman)**

   * Открываем Postman (без долгого обзора интерфейса).

   * Заполняем запрос:

       * Метод: POST

       * URL: /v1/api/users

       * Заголовки: X-Fix-Bug: <твой_ключ>

       * Body (JSON): {"name": "John1", "age": 25}

   * Жмем **Send**.

   * Смотрим результат: 200 OK, юзер создан, ID получен.

   * **Вопрос к студенту:** «Всё ок? Юзер создан. Но стоп... "John123"? Имя с цифрами?»

  

  **Блок 2: Документация — Наш контракт**

   * Как понять, баг это или фича? "Я так чувствую" в IT не работает.

   * Идем в **Документацию** (Swagger).

   * Смотрим правила для поля name: «Только буквы, пробелы и дефисы».

   * **Вывод:** Сервер нарушил собственный контракт. Поздравляю, это твой первый найденный баг бизнес-логики!

  

  **Блок 3: Осмысление — Что такое API? (Без метафор)**

   * В IT любят метафору с официантом. Забудь ее. API умеет управлять домом, запускать ракеты и списывать деньги.

   * Сравниваем интерфейсы (на примере папки в ОС):

       1. **GUI (UI):** Создать папку мышкой (для глаз и рук).

       2. **CLI:** mkdir folder в терминале (для текста).

       3. **API:** Интерфейс для **машин**. Когда программе нужно создать папку, она "нажимает" невидимую кнопку в ОС

          (CreateDir).

   * **Итог:** API — это пульт управления сервером. Мы только что через Postman "нажали кнопку" создания юзера.

  

  **Блок 4: Как нажать кнопку через океан? (HTTP)**

   * Сервер в другой стране. Как наше "нажатие" из Postman до него долетело?

   * Нам нужен курьер — **HTTP (HyperText Transfer Protocol)**.

   * HTTP-запрос — это строгий почтовый конверт.

   * Разбираем конверт на примере того, что мы только что заполнили:

       * **Метод (Method):** Что делаем? (POST — создать)

       * **Endpoint (URL):** Какую кнопку жмем? (/users)

       * **Тело (Body):** Что внутри посылки? ({"name": "John123"})

   * Ответ сервера (Response): тоже конверт, где главное — **Статус-код** (200 OK).

  

  **Блок 5: Клиффхэнгер (Затравка)**

   * Мы запаковали HTTP, дернули API, нашли баг.

   * Но если ты скажешь программисту: "Тут баг!", он ответит: "У меня на компе всё работает".

   * Как доказать, что мы правы? И как выглядит правильный ответ сервера, если он не должен пропускать цифры?

   * Узнаем в следующем уроке (Спойлер: нас ждет Эталон V2).
