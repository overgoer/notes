# 🖥️ Три вида интерфейсов

Представь: нам нужно **создать папку** на Mac/Windows.

---
### 1. GUI (Graphical User Interface)
Интерфейс для глаз и мышки.

- Открываешь Finder / Проводник
- Кликаешь правой кнопкой -> "Создать папку"

---
### 2. CLI (Command Line Interface)
Интерфейс для текста.

- Открываешь Терминал
- Пишешь: `mkdir new_folder`

---
### 3. API (Application Programming Interface)
Интерфейс для ПРОГРАММ.

- Программе не нужна мышка или терминал.
- Она обращается к **API операционной системы**.
- Она "нажимает" системную кнопку: `CreateDirectory()`.

# 🔌 API — это пульт управления

**API** — это голый интерфейс без дизайна. Набор невидимых кнопок, выставленных наружу.

Только нажимают их не пальцы, а другие программы.

Когда мы нажали Send в Postman, мы через API нажали кнопку "создать юзера" на сервере.

# ✉️ Как мы нажали кнопку?

API — это кнопки на сервере в дата-центре. 
Как наше нажатие долетело из Postman до сервера?

Нам нужен курьер. И этот курьер — **HTTP**.

# 📦 HTTP (HyperText Transfer Protocol)

Это строгий "почтовый конверт". <br>
Ты не можешь просто крикнуть серверу. Ты должен запаковать команду в HTTP-запрос.

В конверте 3 главные части, которые мы только что заполняли:

# 🧬 Запрос HTTP
| Часть | Что это? | Как было у нас |

# 🎯 Итог

1. Мы запаковали HTTP-запрос.
2. HTTP-курьер доставил его к API (пульту сервера).
3. API выполнило работу (с багом!) и вернуло HTTP-ответ с кодом `200`.
