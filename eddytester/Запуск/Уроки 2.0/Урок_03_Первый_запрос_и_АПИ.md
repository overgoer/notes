<div align="left">

# 🛑 Осторожно: Метафоры

Часто API объясняют как "официанта в ресторане". 
Ты даешь заказ (меню), он несет его на кухню (сервер) и приносит еду (данные).

**Забудь это.** Метафоры в IT сужают мышление. 
API умеет не только "приносить данные". Оно запускает процессы сборки, списывает деньги с карт, управляет умным домом.

Давай по-взрослому.

---
# 🖥️ Три вида интерфейсов

Представь, что нам нужно **создать папку** на твоем Mac/ПК.

---
### 1. GUI (Graphical User Interface)
Интерфейс для глаз и мышки.

- Ты открываешь Finder / Проводник
- Кликаешь правой кнопкой
- Выбираешь "Создать папку"

---
### 2. CLI (Command Line Interface)
Интерфейс для текста.

- Ты открываешь Терминал
- Пишешь команду: `mkdir new_folder`
- Жмешь Enter

---
### 3. API (Application Programming Interface)
Интерфейс для ПРОГРАММ.

- Если приложению нужно создать папку, оно не двигает мышку и не печатает текст в терминале.
- Оно обращается к **API операционной системы**.
- Оно "нажимает" невидимую кнопку, например: `CreateDirectory()`.

---
<div align="left">

# 🔌 API — это пульт управления

**API** — это голый интерфейс без дизайна. Набор невидимых тумблеров и кнопок, выставленных наружу.

Только нажимают их не пальцы, а другие программы.

Когда ты нажал Send в Postman, ты через API "нажал кнопку" создания юзера на сервере.

---
# ✉️ Как нажать кнопку через океан?

Окей, у сервера в дата-центре есть невидимые кнопки (API). Но как до них дотянуться из твоего дома?

Нам нужен курьер. И этот курьер — **HTTP**.

---
<div align="left" style="font-size: 0.9em">

# 📦 HTTP (HyperText Transfer Protocol)

Это просто формат почтового конверта. Чтобы сервер тебя понял, ты запаковываешь свою команду (нажатие кнопки API) в HTTP-запрос.

В нем всего 3 главные части, которые мы только что заполнили в Postman:

---
# 🧬 Анатомия нашего запроса

| Часть конверта | Что это? | Как было у нас |
|---|---|---|
| **Метод** | Какое действие делаем | `POST` (Создать) |
| **URL (Endpoint)** | Какую кнопку жмем | `.../v1/api/users` |
| **Тело (Body)** | С какими данными | `{"name":"John123"}` |

---
# 🎯 Итог

1. Ты запаковал HTTP-запрос (отдал конверт курьеру).
2. Курьер добежал до API (пульта управления) на сервере.
3. Нажал кнопку "Создать юзера" и передал данные.
4. API выполнило работу (с багом!) и вернуло ответ: `200 OK`.
