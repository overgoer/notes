## Фаза 3 — Эволюция: из реактивной в самоулучшающуюся систему

### Meta-Agent (Supervisor Loop)
- [ ] **Meta-агент (`meta/supervisor.py`)** — запускается после каждого цикла всех агентов
  - Читает все отчёты за цикл → оценивает качество (1-10) по каждому
  - Пишет `meta/report_card.json`: оценки, проблемы, рекомендации
  - Меняет параметры агентов: "Researcher дал 4/10 3 раза → +2 итерации, смени поисковую стратегию"
- [ ] **Hermes Agent** — предлагает улучшения архитектуры по итогам meta-оценок
  - Раз в N дней: "вот что можно улучшить, вот trade-offs, вот приоритет"
  - Формирует задачи в бэклог автоматически

### Память и преемственность
- [ ] **SQLite-память результатов** (`data/agent_memory.db`)
  - `research_log`: тема, ключевые находки, дата, meta-оценка
  - `content_log`: пост, формат, engagement, оценка
  - `search_cache`: запрос → результаты (TTL 7 дней)
- [ ] **Researcher с контекстом** — перед стартом проверяет: "эту тему уже копали → копай глубже / новый угол, не повторяйся"

### Качество контента
- [ ] **Content Quality Gates** — перед отправкой тебе:
  - Uniqueness: не повторяет ли идею из постов за последние 30 дней
  - Depth: у каждого утверждения есть source
  - Tone: соответствует голосу @eddytester
  - Итог: оценка готовности от 0 до 100
- [ ] **Post-mortem после каждого цикла** — "что узнали, что изменить"

### Адаптивное расписание
- [ ] **Agent-to-agent триггеры** — агенты ставят задачи друг другу через `meta/tasks.json`
  - BSA нашёл gap → Researcher запускается вне очереди
  - Meta-agent видит падение качества → меняет параметры
- [ ] **Dynamic iteration count** — Researcher: 3-10 итераций в зависимости от сложности
- [ ] **Early exit** — если всё уже нашли на 3-й итерации, не гонять до 6-й

### Инфраструктура
- [ ] **Researcher: замена DuckDuckGo** — Serper.dev (Google) как primary, SearXNG как fallback
- [ ] **Researcher: self-eval после каждой итерации** — "этот аспект раскрыт → дальше / копай"
- [ ] **Result caching** — не долбить поиск по одинаковым запросам

### Функциональные
- [ ] **Idea Maturation Pipeline** — новые темы в пул → Researcher докидывает контекст раз в неделю → созрела (>3 источников) → в пайплайн
- [ ] **Competitor Drift Detection** — авто-детект изменений стратегии конкурентов
- [ ] **Auto News Brief с контекстом** — "эту новость ты уже освещал, но появились детали: ..."
