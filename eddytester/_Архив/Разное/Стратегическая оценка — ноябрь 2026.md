## Критические пробелы

### 1. Нет дистрибуции

Один пост → один email. TG, Insta, YouTube не получают контент. Это главная проблема: **контент есть, аудитория его не видит**.

- TG 1500 человек не видят постов, которые мы пишем
- Insta 2000 не получает контент
- YouTube 700 не получает контент
- Единственный канал доставки — email (который надо ещё набрать)

### 2. Нет ре repurposing

Один пост может и должен стать:
- Email (полная версия)
- TG пост (сокращённый, с ссылкой «читать полностью»)
- Insta carousel (5-7 слайдов: проблема → шаги → вывод)
- YouTube Short (60 сек: «вот баг, вот фикс»)
- Twitter/X тред (развёрнутый thread)

Сейчас мы делаем только email. Это как иметь двигатель и не поставить колёса.

### 3. Нет монетизации в контенте

Посты учат, но не ведут к продукту. Нет:
- CTA на free-trial-api в конце каждого поста («попробуй сам →`)
- Упоминаний бота в контексте («хочешь ещё 4 таких задачи? → @api_practikum_bot»)
- Прогрева к покупке практикума
- Сегментации контента: бесплатный (привлекает) → прогревающий (строит доверие) → продающий (конвертирует)

### 4. Нет аналитики

Мы не знаем:
- Какие темы заходят лучше
- Какие посты дочитывают
- Где аудитория теряет интерес
- Что конвертит в подписку / покупку
- На чём удвоить, что выкинуть

### 5. YouTube — самый недооценённый актив

700 подписчиков — это мало, но YouTube работает как поисковик. Видео «Как тестировать API: полный гайд 2026» будет находить людей ГОДАМИ. Текстовый пост живёт день. Видео — годы.

У нас есть: экспертиза, контент, сценарии (посты). Нет: продюсера, монтажа, привычки снимать.

## Что превратит тарахтелку в вертолёт-перехватчик

### 1. Multi-Format Engine (самое важное)

Оркестратор пишет пост → **Format Agents** адаптируют под каждую платформу:

```
                ┌─ Email Agent → полный HTML-пост
                │
Пост (MD) ──────┼─ TG Agent → сжатый TG-пост + эмодзи + теги
                │
                ├─ Insta Agent → carousel (5-7 слайдов, текст для каждого)
                │
                ├─ YouTube Agent → сценарий Shorts (60 сек)
                │
                └─ Twitter Agent → тред (10-15 твитов)
```

Один цикл оркестратора = контент на все платформы сразу.

### 2. Monetization Layer

Каждый пост получает невидимую разметку от Monetization Agent:
- **ToF (Top of Funnel):** free-trial API, общие советы — CTA «попробуй сам»
- **MoF (Middle of Funnel):** кейсы, архитектура — CTA на бота с задачами
- **BoF (Bottom of Funnel):** боль, сложность, «как реально научиться» — CTA на практикум

Агент решает: к какой категории относится пост — и добавляет соответствующий CTA.

### 3. Analytics Loop

- Собираем метрики с каждой платформы
- GA получает обратную связь: какие темы залетели, какие нет
- Корректируем пул тем: убираем низкоэффективные, добавляем похожие на успешные

### 4. Product Feedback Loop

Каждый пост → API Scout → бэклог задач практикума.
Каждая новая фича практикума → тема для поста.

Замкнутый цикл: контент питает продукт, продукт питает контент.

> «Мы не хлеб печём — мы делаем контент. Хлеб надо развозить.»

## Принцип: Safety First

TG-канал — главный актив. Одна ошибка ИИ может подорвать доверие подписчиков.
Поэтому дистрибуция идёт ступенями:
1. Draft (ручная проверка) → 2. Одобрение через email (тг 1) → 3. Авто-публикация
Каждая ступень — только когда уверенность 100%.
