## 1. СТАВКИ (BETS) — ПЕРЕСМОТР

### Bet #1: Practicum — единственный P0
**Вердикт:** 🟢 ОСТАВЛЯЕМ. НИЧЕГО НЕ МЕНЯЕТСЯ.

**Почему:** 45 человек сказали «хочу» — 0 заплатили. Free Trial сломан. Платёжки нет. Продукт есть — продаж нет. Всё так же.

**Что докручиваем:**
- Бесплатный триал чиним ПЕРЕД любыми анонсами. Сломанный триал = убитая конверсия в ноль.
- Платёжка — не «потом». Это условие запуска. Без неё Practicum не существует как продукт.

**Метрика:** Выручка Practicum. Цель: первые 10 000₽ до 12 июня.

**Риск:** Технические баги Free Trial оттолкнут первых пользователей. Митигация: починить, протестировать самому, потом анонсировать.

---

### Bet #2: Контент = трибуна для Practicum
**Вердикт:** 🟡 МЕНЯЕМ ПРОПОРЦИИ И ФИЛЬТР.

**Что изменилось:** 44 поста готовы. Датасет проанализирован. Топ-5 по вовлечению — дискуссионные посты и анонсы практикума. Чистые гайды без вопроса = 0 комментов.

**Новая формула:**
- **Старое:** 40/40/20 (вовлечение/обучение/продажа)
- **Новое:** 50/30/20 — больше вовлечения, меньше обучения
- Почему: вовлечение даёт переходы. Обучение без вовлечения = мёртвый контент.

**Фильтр для каждого поста (3 вопроса):**
1. Этот пост заставит остановиться и прочитать?
2. Этот пост вызовет реакцию (коммент/лайк/переход)?
3. Этот пост ведёт к Practicum (прямо или косвенно)?

Если хотя бы на один ответ «нет» — пост не выходит.

### Bet #3: Вовлечение через механику
**Вердикт:** 🟡 ЖЁСТКО ДОКРУЧИВАЕМ.

**Что показали данные (SCOUT + датасет 219 постов):**
- 56% постов с комм — неплохо. 44% без — дохлый контент.
- Топ по вовлечению: анонсы Practicum (45, 25, 17 комм), дискуссии (24, 15).
- Чистые гайды — 0 комментов. Мемы — лотерея.

**Новое правило (жёсткое):** Каждый пост обязан содержать минимум ОДНО из:
1. Вопрос в конце (не «что думаете?», а конкретный)
2. Механику (угадай, исправь, найди, да/нет)
3. Призыв поделиться опытом («кто сталкивался — 👇»)
4. Прямой CTA с низким порогом («➕ если было»)

**Убиваем форматы без вовлечения:**
- ❌ Чистые новости без комментария Эда
- ❌ Ссылки на статьи без своего текста
- ❌ Мемы без вопроса или вовлекающего элемента
- ❌ Посты, где нет вопроса к аудитории

**Метрика:** Доля постов с >5 комм. Цель: >50% (сейчас ~30%).

### Bet #4: Контент-система работает
**Вердикт:** 🔴 МЕНЯЕМ КАРДИНАЛЬНО.

**Проблема:** Система есть — публикаций нет. Ресерчер выдал 17 исследованных тем. 44 поста готовы. Но на канале — 1-2 поста в неделю.

**Новое правило — «Понедельник = День Планирования»:**
- Каждый понедельник: берёшь 3 поста из 44, ставишь в календарь на неделю.
- Никакого «сначала допишу», «сначала перепишу», «сначала улучшу».
- Пост готов = пост публикуется. Точка.

**Что меняем в подходе:**
- **Старое:** «напишу, когда будет идеально»
- **Новое:** «напишу сейчас, исправлю по реакции»

**Ресерчер используем только для:** горячих тем (баг в Twitter, новое в Postman, тренд) — 1 пост в неделю из ресерчера. Остальные 3 — из готовых 44.

**Метрика:** Опубликовано постов в неделю. Цель: 3-4. Факт (последние недели): ~1-2.

## 3. КАЛЕНДАРЬ — СТРУКТУРА

**Файл:** `/root/obsidian-vault/eddytester/Контент/Календарь.md`

### Формат записи:
```
| Дата | Время | Тип | Формат | Тема | CTA | Статус |
```
- **Дата:** ДД.ММ (18.05)
- **Время:** 11:30 или 18:30 МСК
- **Тип:** вовлечение / обучение / продажа
- **Формат:** № из 10 форматов (см. Форматы v2.1)
- **Тема:** кратко о чём пост
- **CTA:** что делаем с аудиторией
- **Статус:** запланирован / написан / опубликован / анализ

### Принципы календаря:
1. **3 поста в неделю:** Пн, Ср, Пт. Не 5 — для 1.5K ядра 5+ убьют retention.
2. **Время:** основные посты в 18:30 (вечер, люди готовы писать). Короткие шпаргалки — в 11:30 (обед).
3. **Распределение по типам:** 2 вовлечения + 1 обучение ИЛИ 1 продажа + 1 вовлечение + 1 обучение.
4. **Каждый пост = номер из 44 готовых.** Исключение: горячая тема от ресерчера (1 раз/нед).
5. **Воскресенье:** аналитика. Какие посты зашли, сколько переходов, корректировка.

## 4. БАНК → ДРУГИЕ ДОМЕНЫ — ПЛАН ПЕРЕРАБОТКИ

✅ Утверждено. Никаких отсылок к банку. Истории из прошлой работы — переработаны под не-банки.

### Новые домены:
1. **E-commerce** — интернет-магазин (корзина, заказы, оплата, остатки). Основной. Самый понятный для джунов.
2. **SaaS** — CRM или платформа (биллинг, тарифы, роли пользователей).
3. **Логистика** — доставка, трекинг, склад.
4. **Геймдев** — игровой API (лидерборды, матчмейкинг, инвентарь).

### Что конкретно меняем:
- **Сериал «Мне дали API банка»** (4 готовые серии) → переписать в «Мне дали API интернет-магазина»
   - Банк → E-commerce платформа
   - Перевод -5000₽ → Цена товара -5000₽
   - Баланс счёта → Остаток на складе
   - Платёжный шлюз → Корзина заказов
   - Security: те же (CORS, rate limiting, авторизация — универсально)
- **Все упоминания «банк» в 44 постах** → заменить на e-commerce / SaaS / конкретный домен
- **Баг-репорты** — контекст остаётся, меняем декорации

### Тайминг:
- Сериал (4 серии) — переписать за 1-2 часа до Пт 22.05 (первая публикация сериала)
- Остальные посты — по мере публикации, перед выходом
