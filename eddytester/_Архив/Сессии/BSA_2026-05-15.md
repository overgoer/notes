## Strategic Bets (3-5 на неделю)

### 1. Интеграция ЮKassa в api-practicum-bot и запуск платного доступа к API Practicum
**Категория:** product | **Effort:** M (2-3 дня на интеграцию, тестирование, деплой)

PM Agent подтвердил GO. Это единственный путь к монетизации — без платежей воронка остаётся разорванной. Необходимо реализовать подписку на v2 API Practicum через бота и Stripe/ЮKassa.

**Ожидаемый эффект:** Появление первого платного продукта, возможность тестировать конверсию, сбор первых денег, замыкание воронки content → попробовать → купить.

**Критерий успеха:** Пользователь может оплатить тестовый период через бота и получить доступ к v2 API Practicum

### 2. Запуск контент-серии «Free Trial: как тестировать бэкенд без денег» вокруг free-trial-api
**Категория:** content | **Effort:** M (1 день на планирование, 2-3 дня на создание контента)

free-trial-api — готовый инструмент для привлечения джунов. Нужно связать его с контентом: сделать 3-4 поста/видео, где показывается, как искать баги в этом API. Это даст рост аудитории и естественный переход к платной версии.

**Ожидаемый эффект:** Рост подписчиков @eddytester, увеличение трафика на API Practicum, повышение осведомлённости о продукте.

**Критерий успеха:** 3 поста в канале, 1 видео/подкаст, >20% новых подписчиков за неделю

### 3. Установка базового мониторинга серверов через Netdata (NO_GO) — замена на лёгкое решение
**Категория:** operations | **Effort:** L (несколько часов: базовый bash-скрипт + cron + Telegram bot)

PM Agent отклонил Netdata из-за сложности, но мониторинг нужен для стабильности продакшен-сервисов. Альтернатива: установить простой скрипт + Telegram-оповещения при падении API или высокой нагрузке. Минимум усилий, максимум пользы.

**Ожидаемый эффект:** Снижение риска незаметного падения продукта, уверенность в uptime, время реакции на сбои.

**Критерий успеха:** Работает уведомление при недоступности портов 80, 3000, 5000 в течение 1 минуты

### 4. Достройка воронки: создание лендинга API Practicum с планом тарифов
**Категория:** product | **Effort:** M (1-2 дня на HTML/CSS, подключение формы подписки или прямой ссылки на оплату)

Без лендинга невозможно объяснить ценность продукта и собрать email-лиды. Нужна простая страница (можно на GitHub Pages или одном файле), где описаны v1 (бесплатно с багами) и v2 (платно, без багов), и кнопка «Получить ключ доступа».

**Ожидаемый эффект:** Увеличение конверсии из канала в продукт, возможность собирать контакты для будущих запусков.

**Критерий успеха:** Лендинг опубликован, нажатие кнопки ведёт к оплате через ЮKassa (после интеграции)
