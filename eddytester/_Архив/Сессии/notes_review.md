### 2. Система управления: Obsidian + 3 ритуала

**Философия:** Не пытаемся сделать всё в одном инструменте. Obsidian — мозг (стратегия + контент + аналитика). Postiz — руки (публикация). PraisonAI — шея (связывает).

#### Ритуал A: ПН — Планирование (30 мин)
1. Открыть `/Календарь/Календарь.md`
2. Выбрать 3 поста из `/Контент/01_Готовые_посты/`
3. Проверить фильтр 3 вопросов
4. Записать в календарь
5. Проверить статус Practicum (Free Trial, платёжка)

#### Ритуал B: Каждый пост — фильтр (5 мин)
1. 3 вопроса: «Остановит? Вызовет реакцию? Ведёт к Practicum?»
2. CTA есть и соответствует цели (вовлечение/продажа)
3. Ссылки работают (проверить на телефоне)

#### Ритуал C: ВС — Ретро (30 мин)
1. Заполнить метрики в `/Аналитика/Еженедельная.md`
2. Какой пост дал макс вовлечения? Почему?
3. Какой пост провалился? Почему?
4. Статус Practicum: триал? платёжка? продажи?
5. Запланировать SCOUT на следующую неделю

### 3. Система хранения постов (связка концептов)

Сейчас 44 поста есть, но они «виртуальные» — живут в форматах. Нужен реестр:

```
01_Готовые_посты/
├── 001_POST_без_тела.md
├── 002_Content_Type_text_plain.md
├── 003_401_Unauthorized_vs.md
├── 004_DELETE_200_vs_204.md
├── 005_Age_150_bug.md
├── ...
└── index.md  ← таблица: номер | формат | тема | CTA | статус
```

`index.md` — единственный источник правды о том, какие посты есть, в каком они состоянии (готов/запланирован/опубликован/архив).

### 4. Что делать с Researcher

**Проблема:** `.venv` в `/root/blog-analysis/agents/` отсутствует.
**Решение:**
1. Создать новый venv: `python3 -m venv /root/blog-analysis/agents/.venv`
2. Установить зависимости: `pip install -r requirements.txt`
3. Протестировать: `run_researcher("парное тестирование в API")`

### 5. Правила игры (чтобы не утонуть)

| Правило | Зачем |
|---------|-------|
| 1. Один источник правды на каждый тип данных | Не дублировать контекст в 5 документах |
| 2. Старые версии → Архив | Не хранить v2..v6 в одной папке с актуальной |
| 3. Пост = 1 файл, не 3 | Не плодить `_v2`, `_апскейл`, `_разбор`, `_финал` |
| 4. Номер поста = номер в index.md | Не «пост про Content-Type», а «пост №002» |
| 5. Каждое ВС — ретро | Без цифр — ты слепой |
| 6. 3 поста/нед — норма | Не 1 глубокий, а 3 любых |
| 7. Researcher = еженедельно | Без свежих тем — канал умирает |
| 8. YouTube = после Practicum | Не распыляться. Сначала 10К выручки |
| 9. Instagram = после 50К выручки | Жёсткое условие. Не обсуждается |
| 10. Если не знаешь что делать → 3 готовых поста | Не думать, публиковать |

### 6. Для YouTube (когда придём)

Когда придёт время YouTube (после Practicum), структура расширяется:

```
eddytester/Контент/
├── 06_YouTube/
│   ├── Скрипты/            ← тексты для видео
│   ├── Снято/              ← готовые видео (ссылки)
│   ├── Идеи/               ← темы для видео
│   └── Плейлисты.md        ← структура плейлистов
```

YouTube не требует отдельной системы. Добавляется как ещё одна категория в `/Контент/`.

### 7. Для Instagram (когда придём)

Instagram — только карусели (Carousel-bot уже есть). Не Reels, не Stories.

```
eddytester/Контент/
├── 07_Instagram/
│   ├── Карусели/           ← PNG + текст
│   ├── Идеи/               ← адаптации TG-постов
│   └── Расписание.md       ← когда и что постить
```

Instagram = перепаковка TG-контента в карусели. Не оригинальный контент.
