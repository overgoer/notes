### Пост 2/10: «Первый баг за 3 минуты»

`POST /api/v1/transfer` — перевод денег между счетами.

Документация: `{ "from": "uuid", "to": "uuid", "amount": 100.50 }`

Отправляю:
```json
{ "from": "alice-uuid", "to": "bob-uuid", "amount": -5000 }
```

Сервер:
```json
{ "status": "success", "transaction_id": "tx-123" }
```

Я перевёл -5000 рублей. Отрицательную сумму. Сервер сказал «успешно».

Для банка — это не баг, это дыра. Ты можешь «отрицательным переводом» накрутить баланс.

Дальше — хуже. Завтра расскажу про авторизацию.

Как думаете — баг или фича? Кидайте версии.

### Пост 7/10: «400 Bad Request с улыбкой»

День четвёртый. Я устал. API ломается каждые 2 запроса.

Делаю `POST /api/v1/pay` с правильным телом. Сервер возвращает:

```json
{ "error": "Bad Request", "message": "Invalid request" }
```

Какое поле невалидное? Не сказано. Что не так? Непонятно.

Звоню разработчику:
— Смотри, 400 на платёж.
— А что в теле?
— Invalid request.
— Ну значит запрос невалидный.
— А что именно?
— Не знаю, смотри код.

Я смотрю код. В response вообще нет `message` — он хардкодом стоит `"Invalid request"` для любого 400.

Ребята, если ваш API возвращает 400 — скажите, **что именно** не так. Не «Invalid request». А `"field: amount, reason: exceeds daily limit"`.

А у вас такое бывает? Кидайте кейсы в комменты.
