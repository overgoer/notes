### Проблемы Amsterdam

1. **🔴 mcp-searxng — 161 процесс-зомби!** Утечка: при каждом рестарте сервиса (systemd `Failed to kill control group`) дочерние процессы не убиваются. Итого 161 экземпляр Node.js, жрущих ~6-7% MEM каждый = почти вся память. ЭТО АВАРИЯ.
2. **🔴 openclaw — 1.6GB RAM + 57% CPU** — это целое ядро процессора и 2ГБ памяти на один гейтвей. На Амстердаме стоит версия 2026.4.29, на Таймвебе 2026.4.24.
3. **🟢 alvin-bot** — 114MB, 0.1% CPU. Лёгкий, но висит без дела (нет активных инстансов TG/WhatsApp/Discord).

### Cherry Studio — вердикт

**На сервере НЕТ установленного Cherry Studio.** То, что лежит в `/tmp/` (35 файлов, ~1.7MB) — это скрипты автоматизации из предыдущих сессий: конфигурация, скриншоты, reset-скрипты. Всё для браузерной автоматизации удалённого Cherry Studio. Никаких сервисов, демонов или systemd-юнитов. Просто временные файлы — можно удалить, не влияет ни на что.

## 3. Оценка агентов

### OpenClaw — вердикт: не оправдывает себя

| Характеристика | Значение |
|---------------|----------|
| RAM на Amsterdam | 1.6GB + 57% CPU |
| RAM на Timeweb | 1.4GB + 24% CPU |
| **Суммарно** | **3GB RAM, ~80% CPU на двух серверах** |
| Функция | Multi-channel AI gateway |
| Версия (AMS) | 2026.4.29 |
| Версия (TW) | 2026.4.24 |

OpenClaw сжирает ресурсы небольшого сервера целиком. Пока не видно чтобы он давал пропорциональную ценность. Возможно, мы используем 10% его возможностей, но платим 100% ресурсов.

### Alvin Bot (v4.22.0) — лёгкий, но без дела

- **RAM:** 114MB — отлично
- **CPU:** 0.1% — незаметен
- **Платформы:** Telegram, WhatsApp, Discord, Signal, Web (CLI)
- **Статус:** Установлен, но не настроен на конкретные каналы. Просто висит.
- **Потенциал:** Если подключить к Telegram как второго бота (для задач/исследований/фоновой работы) — может быть полезен. Но дублирует возможности оркестратора.

### Vibe-Control Agent (Timeweb) — непонятный

- **RAM:** 1.8% (70MB)
- **CPU:** 0%
- **Функция:** Судя по коду — Telegram-бот, который выполняет shell-команды на сервере. Дёшево, но потенциально опасно.
- **Статус:** Работает, используется? Неясно.

### Gemini Bot (Timeweb) — легковесный эксперимент

- Прокси-бот на OpenRouter, использует Gemini 3.1 Pro
- Работает, 1.5% RAM

### Что нужно для нашей архитектуры

Нам не нужен ещё один универсальный AI-агент (Alvin, OpenClaw). Нам нужны **узкоспециализированные агенты под конкретные задачи:**

- **Scout** — читает код практикума, генерирует бэклог (свой код, DeepSeek API) → **ЕСТЬ В ПЛАНЕ**
- **Format Agent** — адаптирует пост под платформу (свой код, DeepSeek API) → **В ПЛАНЕ**
- **Monetization Agent** — ToF/MoF/BoF разметка → **В ПЛАНЕ**

Каждый такой агент — это просто Python-скрипт + DeepSeek API. Не нужны тонны Node.js, не нужно 1.6GB RAM. Наш оркестратор уже доказал что лёгкая архитектура работает.

**Альтернатива OpenClaw:** Если нужен гейтвей для multi-model — можно поднять `litellm` или `openrouter` прокси. Они eat <200MB RAM.
