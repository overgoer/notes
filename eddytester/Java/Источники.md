# 📚 Источники для подготовки Senior QA (Java)

**Принцип:** Используй ресурсы в порядке приоритета. Не пытайся охватить всё сразу.

---

## 🥇 Ядро (обязательно)

### Книги
1. **Хорстманн "Core Java", том 1 (11‑е изд.)**
   - **Для чего:** Фундамент Java, синтаксис, ООП, коллекции.
   - **Главы:** 1–5 (первые 8 недель), потом 6–9.
   - **Формат:** Читай по 1–2 параграфа в день, конспектируй.
   - [Официальный сайт](https://horstmann.com/corejava/)

2. **Блох "Effective Java" (3‑е изд.)**
   - **Для чего:** Лучшие практики, продвинутые темы.
   - **Главы:** 2, 3, 4, 11 (с недели 17).
   - **Формат:** 1–2 пункта в день, применяй в коде.
   - [Amazon](https://www.amazon.com/Effective-Java-3rd-Joshua-Bloch/dp/0134685997)

### Практика
3. **CodingBat**
   - **Для чего:** Мышечная память синтаксиса, базовые алгоритмы.
   - **Разделы:** Warmup‑1, Array‑1, String‑1, Logic‑1.
   - **Режим:** 3–5 задач/день без IDE.
   - [codingbat.com/java](https://codingbat.com/java)

4. **LeetCode**
   - **Для чего:** Алгоритмы для интервью, Big O, структуры данных.
   - **Подборка:** [Top 100 Liked](https://leetcode.com/problem-list/top-100-liked-questions/)
   - **Режим:** С недели 9, 1–2 задачи Easy/день.
   - [leetcode.com](https://leetcode.com/)

5. **Neetcode.io**
   - **Для чего:** Структурированный путь по алгоритмам.
   - **Порядок:** Arrays & Hashing → Two Pointers → Sliding Window → ...
   - [neetcode.io](https://neetcode.io/)

## 🥈 Дополнительно (когда база есть)

### Java углублённо
6. **Java Concurrency in Practice** (Brian Goetz)
   - **Для чего:** Многопоточность (с недели 17).
   - [Amazon](https://www.amazon.com/Java-Concurrency-Practice-Brian-Goetz/dp/0321349601)

7. **"Философия Java"** (Брюс Эккель)
   - **Для чего:** Понимание ООП, паттернов.
   - [Thinking in Java](https://www.mindviewllc.com/books/thinking-in-java/)

### QA специфика
8. **JUnit 5 User Guide**
   - **Для чего:** Написание unit‑тестов.
   - [junit.org/junit5/docs/current/user-guide](https://junit.org/junit5/docs/current/user-guide/)

9. **Mockito Documentation**
   - **Для чего:** Мокирование зависимостей.
   - [site.mockito.org](https://site.mockito.org/)

10. **TestContainers**
    - **Для чего:** Интеграционные тесты с БД, Docker.
    - [testcontainers.org](https://testcontainers.org/)

11. **REST Assured**
    - **Для чего:** Тестирование REST API.
    - [rest-assured.io](https://rest-assured.io/)

12. **Selenide**
    - **Для чего:** UI‑тестирование (лаконичный синтаксис).
    - [selenide.org](https://selenide.org/)
